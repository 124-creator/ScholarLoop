可信论文搜索 Agent
不是给列表，而是给证据链

ScholarLoop 面向“科研场景下复杂学术查询的智能论文搜索与推荐”。它把复杂问题拆成可检索子问题，用 BM25、Embedding、Cross-Encoder 做融合排序，再把每条推荐落到可点击核验的 evidence card 和 source span。

这不是普通 RAG Demo，而是比赛评分导向的系统

本系统围绕赛题要求展开：官方 F1、效率、结构化展示与专家评审同时存在，所以页面必须证明“能检索、能解释、能核验、能交付”。

公开/隐藏测试压力

页面叙事不再只说“模型很强”，而是把 LitSearch、RealScholarQuery/PaSa、BM25 对照和 ΔF1 放到可核查指标区。

复杂查询理解

比赛题目要求复杂学术查询。Demo 把任意研究问题先拆成对象、方法、数据、指标、争议点，再进入多路召回。

可信推荐

最终不是“推荐 10 篇论文”这么简单，而是每条结论必须有 evidence card、source_field、char_span 和人工核验边界。

双 Loop：一个追求召回，一个防止编造

Search Loop 负责把复杂问题转成候选集合；Trust Loop 负责把候选理由压回原文证据。每一轮都有可交接文档，不靠一句“模型认为”。

Loop 1 · Search Loop

1
query decomposition：拆解研究对象、方法、数据、指标。
handoff.query_plan.json
2
hybrid retrieval：BM25 + Dense + sub-query recall 多路召回。
handoff.candidates.jsonl
3
rerank fusion：Cross-Encoder 与权重融合，输出可复现排序报告。
handoff.ranking_result.json

Loop 2 · Trust Loop

4
evidence landing：每个推荐字段绑定 source_field 和 char_span。
handoff.evidence_card.json
5
click-to-verify：只有 source_text[char_span] == field value 才允许高亮。
handoff.span_fidelity.json
6
audit trail：每一步说明证据来源、边界和人工审查点。
handoff.trace_report.json

交互演示：每一步都留下交付物

点击流程节点会切换当前交接物。这个公开页不调用私有语料、密钥或实时 LLM；它展示的是已通过离线验证的可信检索链路如何组织成可公开演示的产品体验。
