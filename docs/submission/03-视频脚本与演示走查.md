# 项目视频脚本与演示走查

> 本文件只提供录制脚本与分镜；实际录制由人工完成。所有展示值来自 verified JSON，不现场联网、不调用 LLM。

## 0. 开场（约 20 秒）

画面：标题页 + 单命令启动终端。

旁白：ScholarLoop 面向复杂学术查询，目标是在公开基准 F1、运行效率、结构化展示与专家创新维度之间取得平衡。Demo 采用 offline 0-LLM 路径，页面所有数值来自 verified 产物。

证据引用：`reports/m080/fidelity_audit.json`。

## 1. 单命令启动（约 20 秒）

操作：

```powershell
$env:PYTHONPATH="src"
python -m scholarloop.demo.app --host 127.0.0.1 --port 8766
```

浏览器打开 `http://127.0.0.1:8766`。

讲解点：这是整合层，不改 M030 web 基线；offline、0 LLM/request。引用：`src/scholarloop/demo/app.py`、`reports/m080/smoke.txt`。

## 2. 查询理解与拆解（约 40 秒）

操作：选择 `litsearch_000`。

讲解点：复杂查询被拆成多个检索条件，Demo 直接读取 M040 的 `decomposition_from_m010`，不重新生成。引用：`reports/m040/results.json.per_query`。

## 3. A-v2 排序面板（约 60 秒）

操作：展示 top20 排序、gold 命中标记、M040 reason。

讲解点：A-v2 在 LitSearch F1=`0.1311870491`，高于 BM25 `0.0963937793`；RealScholarQuery F1=`0.1972078900`，高于 BM25 `0.1058441087`。引用：`reports/m040/results.json.by_system`、`reports/m060/results.json.metrics.resolvable.aggregate`。

## 4. 证据矩阵（约 60 秒）

操作：滚动 B-lite 矩阵，展示 source_field、char_span、证据状态。

讲解点：每个字段来自 M020 evidence JSON；缺失或不足不补写，保持“已有证据支持/证据不足/存在争议/需人工核验”的状态。引用：`reports/m020/evidence/*.json`。

## 5. 真实出处富化（约 50 秒）

操作：展示作者/年份/DOI/出处列。

讲解点：M050 用 OpenAlex 等开放元数据连接器富化；提交演示读取 `enriched_replay` 离线快照，未解析字段只标“需人工核验”。引用：`reports/m050/enriched_replay/*.json`、`reports/m050/data-sources.md`。

## 6. 研究空白候选启发（约 60 秒）

操作：展示空白候选列表与关系矩阵边。

讲解点：M070 提供“高活跃·零历史共现”概念对的候选生成启发，候选填补率 `0.108`，原随机基线 `0.022`。M100 频率配平消融进一步显示，频率匹配随机对为 `0.306`、频率配平 Δ=`-0.198`，因此这里不把研究空白作为频率无关的独立预测能力，只作为候选方向提示和自我证伪案例。引用：`reports/m070/significance.json`、`reports/m070/gaps_display.json`、`reports/m100/gap_frequency_ablation.json`。

## 7. 收束（约 30 秒）

讲解点：整合校验显示 fabrication=`0`、out_of_pool=`0`，提交物料每条指标均挂 verified 来源，不扩展未验证主张。引用：`reports/m080/fidelity_audit.json`、`docs/submission/08-主张溯源清单.md`。
