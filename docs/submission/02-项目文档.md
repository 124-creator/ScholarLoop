# ScholarLoop 项目文档（提交版）

## 1. 问题与目标

赛题要求端到端处理复杂学术查询，覆盖查询理解与分解、自主搜索策略优化、论文综合排序、搜索结果结构化归纳四项核心功能；评分关注 F1、运行效率和结构化展示，专家评审关注创新、落地和泛化。事实源：`00-赛题/科研场景下复杂学术查询的智能论文搜索与推荐.txt`、`docs/dev/spec/015-成功标准校准.md`。

ScholarLoop 的目标是把“高 F1 排序”和“可复核证据链”同时落地：推荐列表必须来自公开语料候选池，证据字段必须能回到原文或外部元数据快照，缺失字段只提示人工核验，不补写。边界来源：`reports/m020/verification.json`、`reports/m050/data-sources.md`。

## 2. 方法概览

1. **查询理解与拆解**：复用 M010/M040 已落盘的查询拆解字段，Demo 不再调用 LLM。来源：`reports/m040/results.json.per_query[*].decomposition_from_m010`。
2. **多路召回与排序**：A-v2 融合 BM25、dense_v2、子查询信号与 cross-encoder 重排，最终权重来自 train split，holdout/test 只报告。实测 dense 模型为 `BAAI/bge-small-en-v1.5`，不是 BGE-base。来源：`reports/m040/results.json.protocol`。
3. **证据矩阵**：B-lite 对每条推荐展示字段、状态、source_field 与 char_span。来源：`reports/m020/evidence/*.json`。
4. **真实元数据连接器**：OpenAlex 主源补作者/年份/DOI；未解析时保留“需人工核验”。来源：`reports/m050/enriched_replay/*.json`、`reports/m050/data-sources.md`。
5. **研究空白候选生成启发**：以确定性概念对检测 + 时间切片回看输出“高活跃、零历史共现”的候选方向，LLM 只做接地叙述且越界论文 id 为 0。M100 频率配平消融显示旧预测优势未保持，因此本项不声称频率无关的独立预测能力。来源：`reports/m070/results.json`、`reports/m070/gaps_display.json`、`reports/m100/gap_frequency_ablation.json`。

## 3. 系统架构

```text
官方/公开基准与缓存
  ├─ LitSearch / RealScholarQuery verified JSON
  ├─ M040 A-v2 排序结果
  ├─ M020 证据矩阵
  ├─ M050 enriched_replay 元数据快照
  └─ M070 研究空白候选启发结果
        ↓ 只读装配
src/scholarloop/demo/assemble.py
        ↓
src/scholarloop/demo/app.py  (offline HTTP demo, 0 LLM/request)
        ↓
四层面板：查询拆解 / A-v2 排序 / 证据矩阵 / 真实出处 / 研究空白候选启发
```

整合层是新增 `src/scholarloop/demo/**`，不改 M030 `src/scholarloop/web/**` 与任何 M010–M070 产物。核验来源：`reports/m080/fidelity_audit.json`。

## 6. 创新与边界

- 创新主张限定为：A-v2 跨两公开基准显著优于 BM25；证据矩阵与真实连接器降低虚构风险；研究空白只作为“高活跃·零历史共现”候选生成启发。M100 频率配平消融显示，旧的未来填补率优势主要由概念频率驱动，提交表述不再把它作为独立预测能力。来源：`reports/m040/results.json`、`reports/m060/significance.json`、`reports/m070/significance.json`、`reports/m100/gap_frequency_ablation.json`。
- M100 将频率混杂正面检验并保留负结果：候选填补率 `0.108` 低于频率匹配随机对 `0.306`，频率配平 Δ=`-0.198`。该自我证伪过程是可信治理证据，不是隐藏项。来源：`reports/m100/gap_frequency_ablation.json`、`docs/dev/000-决策日志.md` M100/M110 记录。
- M040 cross-encoder 在 LitSearch 上相对 no-rerank 的 F1 增益约 `0.003137`，NDCG@20 增益约 `0.020425`；可保留为排序质量细化件，但不夸大为主要 F1 来源。来源：`reports/m040/results.json.by_system`。
- M060 第二基准报告 A-v2 与 no-rerank 行，但未把 cross-encoder 单独作为第二基准消融承重。来源：`reports/m060/results.json.metrics`。
