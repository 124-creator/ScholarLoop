## 一句话目标

构建一个面向研究生科研场景的 Web 智能系统，使用户输入复杂学术问题后，系统能够自动完成问题拆解、多轮学术检索、论文筛选、证据归因、综合排序与研究空白总结，最终输出可用于开题、综述写作、选题判断和创新点发现的结构化科研证据链。

---

## 真正要解决的问题

研究生在真实科研过程中遇到的核心困难，并不是“找不到论文”，而是：

1. 不知道如何把复杂科研问题拆解成可检索的问题；
2. 不知道哪些论文真正相关、可靠、前沿；
3. 不知道不同论文之间的方法、数据、结论和局限如何对应；
4. 不知道某个方向是否还有可发表的研究空白；
5. 容易被高引用论文、关键词匹配或大模型幻觉误导；
6. 难以快速形成可复核、可引用、可继续扩展的科研证据链。

因此，本项目要解决的不是普通论文搜索问题，而是：

> 面向复杂科研问题的可信学术检索、证据组织与研究空白发现问题。

---

## 目标用户

1. 研究生、低年级科研人员；
2. 需要完成开题、综述、论文选题、方法比较的科研用户；
3. 数学建模、人工智能、数据科学、交叉学科竞赛团队；
4. 需要快速判断一个研究方向是否值得继续做的个人研究者。

---

## 核心场景

用户输入复杂科研问题，例如：

> “严格预测协议下，碳价预测中外生变量为什么经常失效？有哪些可解释模型和验证方法？”

系统需要完成以下闭环：

1. 自动识别研究对象、方法、数据、指标、争议点和应用场景；
2. 将复杂问题拆解为多个可检索子问题；
3. 调用公开学术数据库或本地论文库进行多轮检索；
4. 对论文进行去重、过滤、相关性判断和证据提取；
5. 按相关性、权威性、新颖性、证据强度、多样性进行综合排序；
6. 输出推荐论文清单、证据矩阵、研究图谱和研究空白分析；
7. 明确标注证据来源、不确定性和需要人工核验的结论。

---

## 成功标准

### 1. 系统功能成功标准

系统至少完成以下 6 个核心模块：

1. 复杂学术问题输入模块；
2. 查询拆解与检索意图识别模块；
3. 多源学术检索模块；
4. 论文过滤与综合排序模块；
5. 证据矩阵与引用归因模块；
6. 研究空白与后续选题建议模块。

验收方式：

* 能够在 Web 页面中完整演示；
* 至少支持 3 个完整案例；
* 每个案例能够输出结构化结果，而不是只返回论文列表。

---

### 2. 检索推荐效果成功标准

构建一个小型评测集，至少包含 50 个复杂学术查询问题。

每个问题至少标注：

1. 查询意图；
2. 推荐论文；
3. 相关性等级；
4. 方法标签；
5. 数据标签；
6. 结论标签；
7. 是否存在研究空白。

与以下基线方法对比：

1. 普通关键词搜索；
2. BM25；
3. Embedding Search；
4. 单轮 LLM 检索；
5. ScholarLoop 完整流程。

至少在以下指标中取得明显优势：

1. Precision@10；
2. Recall@20；
3. F1；
4. NDCG；
5. 人工评价得分；
6. 证据归因准确率。

最低验收阈值：

* ScholarLoop 在至少 2 个核心指标上优于基线；
* 人工抽检推荐结果合理率达到 85% 以上；
* 不允许出现虚构论文、虚构 DOI、虚构作者、虚构期刊。

---

### 3. 证据链成功标准

每篇推荐论文必须输出：

1. 论文标题；
2. 作者与年份；
3. 来源或 DOI；
4. 推荐理由；
5. 支持的研究问题；
6. 使用的方法；
7. 使用的数据或实验场景；
8. 主要结论；
9. 局限性；
10. 与用户问题的关联强度。

系统必须显式区分：

1. 已有证据支持；
2. 证据不足；
3. 存在争议；
4. 需要人工核验。

---

### 4. 创新性成功标准

项目不能停留在普通 RAG 或论文搜索网页，而必须体现以下创新点：

1. 面向复杂科研问题的查询拆解机制；
2. 多轮检索与自我修正机制；
3. 相关性、权威性、新颖性、证据强度、多样性的综合排序机制；
4. 论文推荐结果的证据矩阵化组织；
5. 研究空白发现与后续选题建议；
6. 面向研究生科研流程的 Web 化闭环演示；
7. 可复现评测集、基线对比和消融实验。

---

### 5. 比赛展示成功标准

最终提交材料至少包括：

1. 可运行 Web Demo；
2. 项目技术报告；
3. 系统架构图；
4. 核心算法流程图；
5. 评测集说明；
6. 对比实验表；
7. 消融实验表；
8. 至少 3 个完整演示案例；
9. 演示视频；
10. 项目 README；
11. 核心代码；
12. 风险与局限性说明。

一等奖标准不是“网页好看”，而是：

> 系统完整、问题真实、技术闭环、实验可验证、演示有冲击力、创新点讲得清楚。

---

## 技术形态

最终系统优先采用 Web 形态。

原因：

1. 企业赛题和人工智能创新大赛都更容易通过 Web 系统展示完整能力；
2. Web 页面更适合演示复杂查询、检索过程、证据矩阵和研究图谱；
3. 个人参赛条件下，Web 系统比机器人、硬件平台、视频生成系统更可控；
4. Web Demo 便于录制视频、截图展示和答辩讲解。

Web 系统不是项目创新本身，而是项目展示载体。
真正创新点必须体现在复杂问题拆解、检索推荐、证据归因和研究空白发现上。

---

## 案例策略

碳价格预测可以作为系统的一个演示案例，但不能作为唯一核心案例。

原因：

1. 碳价预测实验结果目前并不突出；
2. 如果把项目完全绑定到碳价预测，容易被限制在单一应用领域；
3. 本项目要体现的是通用科研智能体能力，而不是某一篇未发表论文的结果；
4. 碳价预测可以用于展示系统处理“复杂、争议性、证据不足的科研问题”的能力。

建议最终使用 3 类案例：

1. 用户熟悉案例：碳价预测、时间序列预测、外生变量失效；
2. 热点 AI 案例：RAG、Agent、AI for Science、科研智能体；
3. 跨学科案例：医学影像、智能制造、环境预测、灾害预警等。

## 硬约束

### 1. 个人参赛约束

由于是个人参赛，必须控制工程范围。

优先保证：

1. 核心流程可运行；
2. 演示案例完整；
3. 实验结果可信；
4. 报告叙事清楚；
5. 页面足够专业。

不追求：

1. 大规模分布式系统；
2. 大模型训练；
3. 复杂硬件部署；
4. 全自动科研替代；
5. 过度炫技的前端动画。

---

### 2. 技术约束

1. 优先使用公开数据库、公开 API 或本地可复现论文库；
2. 可以使用大模型做查询拆解、摘要理解、证据组织和结果解释；
3. 检索、排序、评测逻辑必须尽量可解释；
4. 不依赖昂贵 GPU；
5. 不依赖不可公开数据；
6. 不依赖无法复现的闭源流程。

---

### 3. 数据约束

1. 使用公开论文元数据、摘要、引用信息或开放全文；
2. 不使用未经授权的付费全文数据；
3. 所有评测样例、检索结果、人工标注和实验结果必须保存；
4. 所有核心结果需要可复查、可复现。

---

### 4. 合规约束

1. 不生成虚假引用；
2. 不伪造 DOI、作者、期刊、年份；
3. 不伪造实验结果；
4. 不宣称系统可以完全替代人工科研判断；
5. 对不确定结论必须显式标注；
6. 对低置信度推荐必须提示人工核验。

---

## 明确不做

1. 不做通用聊天机器人；
2. 不做普通关键词论文搜索网站；
3. 不做只返回论文标题列表的推荐工具；
4. 不做没有评测集、没有基线对比的展示项目；
5. 不做依赖大规模 GPU 训练的大模型微调项目；
6. 不优先做机器人、视频扩散、工业质检、NPU 底层优化等与当前个人资源不匹配的方向；
7. 不把碳价预测作为唯一核心场景；
8. 不做无法解释推荐理由的黑箱系统；
9. 不做只追求页面好看但缺少科研证据链的项目。

---

## 已知事实

1. 参赛者为个人参赛；
2. 参赛者具备前端、后端和公开数据库使用能力；
3. 参赛目标为进入决赛并冲击一等奖；
4. 已确定选择“科研场景下复杂学术查询的智能论文搜索与推荐”方向；
5. 项目最终形态倾向于 Web 系统；
6. 参赛者已有多智能体工作流、科研文献分析、数学建模、碳价预测研究和项目治理经验；
7. 碳价预测案例可以作为演示案例，但不作为唯一核心创新来源；
8. 项目必须比普通 RAG 网页更有创新性和可验证性。

---

## 未知项

1. 企业赛题最终是否提供隐藏测试集；
2. 是否有官方指定数据源、接口或评测脚本；
3. 最终提交材料是否有页数、视频时长、代码格式限制；
4. 可调用外部 API 的额度和稳定性；
5. 最终答辩环节对系统实时演示的要求；
6. 评委更偏重算法指标、工程完整度还是应用价值。

---

## 初步技术路线

1. 问题理解层

   * 复杂学术查询解析；
   * 研究对象、方法、数据、指标、争议点识别；
   * 查询拆解和查询改写。

2. 检索层

   * 关键词检索；
   * 向量检索；
   * 学术 API 检索；
   * 本地论文库检索；
   * 多轮检索与结果合并。

3. 过滤层

   * 去重；
   * 低质量论文过滤；
   * 相关性初筛；
   * 年份、来源、引用、开放性筛选。

4. 排序层

   * 相关性评分；
   * 权威性评分；
   * 新颖性评分；
   * 证据强度评分；
   * 多样性约束；
   * 综合排序。

5. 证据层

   * 摘要证据提取；
   * 方法标签；
   * 数据标签；
   * 结论标签；
   * 局限性标签；
   * 引用与来源归因。

6. 输出层

   * 推荐论文清单；
   * 证据矩阵；
   * 研究主题图谱；
   * 研究空白总结；
   * 后续选题建议。

7. 评测层

   * 构建复杂查询评测集；
   * 设计人工标注规则；
   * 与关键词搜索、BM25、Embedding Search、单轮 LLM 检索对比；
   * 做消融实验；
   * 做案例分析和失败案例分析。

## 冻结结论

本项目正式锁定为：

> 面向研究生复杂科研问题的可信论文搜索、证据链组织与研究空白发现智能体。

项目不再围绕“是否选择赛题”进行反复摇摆。
后续所有设计、开发、实验和展示，均必须服务于该问题定义。

若后续需要改变核心目标、赛题方向、系统形态或一等奖标准，必须回到本文件重新冻结，并在 `docs/dev/000-决策日志.md` 中追加变更记录。
