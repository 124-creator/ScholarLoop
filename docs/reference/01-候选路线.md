## 0. 冻结边界复述

本文件只在方案层发散候选路线，不选择最终路线，不展开业务实现细节，不改变 `010-问题定义.md` 中已经冻结的目标、成功标准、硬约束和 out of scope。

### 0.1 冻结目标

项目锁定为：

> 面向研究生复杂科研问题的可信论文搜索、证据链组织与研究空白发现智能体。

系统形态优先为 Web 系统。用户输入复杂学术问题后，系统需要形成从问题拆解、多轮检索、论文筛选、证据归因、综合排序到研究空白总结的闭环，最终输出可用于开题、综述写作、选题判断和创新点发现的结构化科研证据链。

### 0.2 冻结成功标准摘要

候选路线必须服务于以下成功标准：

1. **系统功能完整性**：至少覆盖复杂学术问题输入、查询拆解与检索意图识别、多源学术检索、论文过滤与综合排序、证据矩阵与引用归因、研究空白与后续选题建议 6 个核心模块；能够在 Web 页面中完整演示；至少支持 3 个完整案例；每个案例输出结构化结果。
2. **检索推荐效果**：构建至少 50 个复杂学术查询问题的小型评测集，并与普通关键词搜索、BM25、Embedding Search、单轮 LLM 检索、ScholarLoop 完整流程对比；至少在 2 个核心指标上优于基线；人工抽检推荐结果合理率达到 85% 以上；不允许出现虚构论文、虚构 DOI、虚构作者、虚构期刊。
3. **证据链完整性**：每篇推荐论文必须输出标题、作者与年份、来源或 DOI、推荐理由、支持的研究问题、方法、数据或实验场景、主要结论、局限性、与用户问题的关联强度；系统必须显式区分已有证据支持、证据不足、存在争议、需要人工核验。
4. **创新性**：不能停留在普通 RAG 或论文搜索网页，必须体现复杂问题查询拆解、多轮检索与自我修正、综合排序、证据矩阵化组织、研究空白发现、Web 化科研流程闭环、可复现评测集与基线/消融实验。
5. **比赛展示完整性**：最终需要支撑可运行 Web Demo、技术报告、系统架构图、核心算法流程图、评测集说明、对比实验表、消融实验表、至少 3 个完整演示案例、演示视频、README、核心代码、风险与局限性说明。

### 0.3 冻结硬约束与不做事项摘要

所有候选路线必须满足：

1. 个人参赛，工程范围必须可控，优先保证核心流程可运行、演示案例完整、实验结果可信、报告叙事清楚、页面足够专业。
2. 优先使用公开数据库、公开 API 或本地可复现论文库；允许使用大模型做查询拆解、摘要理解、证据组织和结果解释；检索、排序、评测逻辑必须尽量可解释。
3. 不依赖昂贵 GPU、不依赖不可公开数据、不依赖无法复现的闭源流程。
4. 使用公开论文元数据、摘要、引用信息或开放全文；不使用未经授权的付费全文数据。
5. 所有评测样例、检索结果、人工标注和实验结果必须保存，核心结果需要可复查、可复现。
6. 不生成虚假引用，不伪造 DOI、作者、期刊、年份，不伪造实验结果，不宣称系统可完全替代人工科研判断。
7. 不做通用聊天机器人，不做普通关键词论文搜索网站，不做只返回论文标题列表的推荐工具，不做没有评测集和基线对比的展示项目，不做大规模 GPU 微调项目，不把碳价预测作为唯一核心场景，不做无法解释推荐理由的黑箱系统。

## 1. 候选路线总览

| 路线 | 路线名称 | 差异化取向 | 最适合强化的成功标准 |
|---|---|---|---|
| A | 评测驱动的可解释检索排序路线 | 以可解释评分、可复现评测和稳定输出为中心 | 检索推荐指标、证据归因准确率、个人可控交付 |
| B | 证据图谱驱动的研究空白发现路线 | 以论文—方法—数据—结论—局限的结构化图谱为中心 | 证据矩阵、研究图谱、研究空白发现、展示冲击力 |
| C | 审稿式多智能体自我修正路线 | 以多轮规划、检索、质检和纠错过程为中心 | 多轮检索与自我修正、智能体特征、复杂问题闭环演示 |

三条路线均保持同一个冻结目标，但侧重点不同：路线 A 偏“稳健可验证”，路线 B 偏“结构化知识组织与空白发现”，路线 C 偏“智能体过程与动态修正”。本文件不对三者排序，也不选定最终路线。

## 2. 路线 A：评测驱动的可解释检索排序路线

### 2.1 核心思想

以“复杂问题拆解 + 多源候选召回 + 可解释综合排序 + 证据矩阵输出”为主线，优先把系统做成一个稳定、可复查、可评测的科研检索推荐流水线。

这条路线的核心不是追求最复杂的智能体形态，而是把 ScholarLoop 的优势集中在以下方面：

1. 将复杂学术问题拆成研究对象、方法、数据、指标、争议点、应用场景等检索意图；
2. 用多种检索通道获得候选论文，避免单一关键词或单一向量召回的偏差；
3. 对候选论文按相关性、权威性、新颖性、证据强度、多样性做显式评分；
4. 把推荐结果组织为证据矩阵，而不是论文标题列表；
5. 以 50 个复杂查询评测集、基线对比和消融实验证明完整流程优于普通搜索、BM25、Embedding Search 和单轮 LLM 检索。

### 2.2 关键假设

1. 公开论文元数据、摘要、引用信息和开放全文片段，足以支撑大多数演示案例中的相关性判断、证据归因和局限性总结。
2. 通过显式的查询拆解和综合排序，可以在 Precision@10、NDCG、人工评价得分、证据归因准确率等指标中至少获得 2 项相对基线的明显优势。
3. 小型但高质量的 50 问评测集能够体现系统处理复杂科研问题的能力，而不会被评委认为只是少量定制案例。
4. 个人参赛条件下，优先做稳定可复现的排序流水线，比追求复杂但不稳定的全自动智能体更容易达成最低验收阈值。

### 2.3 预期收益

1. **交付稳定性高**：路线结构清晰，容易形成可运行 Web Demo、技术报告、架构图、流程图、评测集说明、对比实验表和消融实验表。
2. **评测闭环强**：天然围绕 50 问评测集和多基线对比组织，容易证明 ScholarLoop 完整流程在至少 2 个核心指标上优于基线。
3. **合规风险较低**：显式引用来源、DOI、作者、年份和人工核验提示，有利于避免虚构引用和不可复查结论。
4. **个人工程可控**：不依赖昂贵 GPU，不需要大规模训练，不需要复杂分布式系统，符合个人参赛约束。
5. **叙事清楚**：可以把项目讲成“复杂科研问题不是直接搜索，而是拆解、召回、排序、归因、评测的可信闭环”。

### 2.4 主要风险

1. **创新感可能不足**：如果查询拆解、综合排序和证据矩阵表现不突出，容易被看成“增强版论文搜索/RAG”。
2. **研究空白发现可能偏弱**：流水线式排序擅长找相关论文，但对“可发表研究空白”的识别可能更依赖规则化总结，冲击力不如图谱或智能体过程。
3. **指标优势受评测集影响大**：如果 50 问评测集设计不够复杂或标注质量不稳定，实验说服力会下降。
4. **证据强度评分容易被质疑**：相关性、权威性、新颖性、证据强度、多样性等评分需要解释清楚，否则会被认为是主观打分。

### 2.5 时间与资源成本

| 维度 | 成本判断 |
|---|---|
| 时间成本 | 中。需要完成完整流程、评测集、基线对比、消融实验和 3 个案例，但路线边界清晰，可分阶段推进。 |
| 工程成本 | 中。主要成本在多源检索接入、候选合并、排序解释、证据矩阵展示和评测记录保存。 |
| 数据成本 | 中。需要人工构建和维护至少 50 个复杂查询问题及其标注结果。 |
| 算力成本 | 低。不依赖昂贵 GPU，可优先使用公开数据库、公开 API、本地索引和大模型 API。 |
| 展示成本 | 中。Web 页面重点展示拆解过程、排序理由、证据矩阵、基线对比和案例输出，不追求重动画。 |

### 2.6 不确定度

**中。**

路线 A 的主要能力边界较清楚，个人可控性较强，但能否在创新性和研究空白发现上达到一等奖级别仍存在不确定性。

### 2.7 最危险假设

**公开元数据、摘要和开放全文片段足以支撑可信证据归因与排序优势。**

如果这个假设不成立，系统可能只能输出看似合理但证据不足的推荐理由，难以达到证据链成功标准，也难以在证据归因准确率和人工评价得分上明显超过基线。

### 2.8 与冻结成功标准的对应关系

| 冻结成功标准 | 路线 A 的对应方式 |
|---|---|
| 6 个核心功能模块 | 以输入、查询拆解、多源检索、过滤排序、证据矩阵、研究空白建议构成主流水线。 |
| Web 完整演示与 3 个案例 | 适合在 Web 中展示从问题拆解到推荐排序再到证据矩阵的稳定流程；案例可覆盖碳价预测、AI 科研智能体、跨学科预测/医学影像等。 |
| 50 问评测集与基线对比 | 路线核心即评测驱动，天然支持普通关键词搜索、BM25、Embedding Search、单轮 LLM 检索和完整流程对比。 |
| 至少 2 个核心指标优于基线 | 重点争取 Precision@10、NDCG、人工评价得分、证据归因准确率。 |
| 不允许虚构引用 | 通过来源字段、DOI/来源校验、低置信度人工核验提示降低风险。 |
| 每篇论文 10 项证据字段 | 证据矩阵作为主输出，可覆盖标题、作者年份、来源/DOI、推荐理由、问题关联、方法、数据、结论、局限性、关联强度。 |
| 显式区分证据状态 | 可在证据矩阵中设置已有证据支持、证据不足、存在争议、需要人工核验等状态。 |
| 创新性 | 主要体现为复杂查询拆解、可解释综合排序、证据矩阵、评测集和消融实验。 |
| 个人参赛硬约束 | 工程范围相对可控，不依赖大规模训练、昂贵 GPU 或不可公开数据。 |
| 明确不做事项 | 不做通用聊天机器人，不做普通标题列表搜索，不做无评测展示，不做黑箱推荐。 |

## 3. 路线 B：证据图谱驱动的研究空白发现路线

### 3.1 核心思想

以“研究问题—论文—方法—数据—指标—结论—局限—争议点”的结构化证据图谱为核心，把 ScholarLoop 做成一个能够组织科研证据、显示知识覆盖关系、辅助发现研究空白的 Web 智能系统。

这条路线的重点不是先把论文排成一个列表，而是先把检索到的论文转化为可比较的证据单元，再通过图谱关系和矩阵空缺识别：

1. 哪些方法在某类数据或场景中被反复验证；
2. 哪些结论存在冲突或边界条件；
3. 哪些数据、指标、实验场景或验证协议缺少覆盖；
4. 哪些论文只支持局部结论，不能外推到用户问题；
5. 哪些方向可能形成后续选题或创新点。

### 3.2 关键假设

1. 不同学科案例可以被抽象到一套相对统一的证据结构中，例如研究对象、方法、数据、指标、结论、局限、争议和空白。
2. 结构化证据图谱比单纯排序列表更能体现“可信证据链”和“研究空白发现”的创新性。
3. 公开摘要、元数据、引用信息和开放全文片段中包含足够信息，可以支撑方法、数据、结论、局限等图谱要素抽取。
4. 评委能够理解并认可“图谱覆盖缺口、证据冲突、低覆盖组合”作为研究空白发现的可解释依据。
5. 个人参赛者可以控制图谱规模，优先围绕 3 个完整案例和 50 问评测集做小而精的可复现图谱，而不是追求大规模知识库。

### 3.3 预期收益

1. **创新表达强**：相比普通 RAG 或搜索排序，证据图谱更容易展示“论文之间的方法、数据、结论和局限如何对应”。
2. **研究空白发现更有抓手**：可以通过图谱中的空缺、弱连接、冲突边、低证据强度区域来解释为什么某个方向仍有研究空间。
3. **展示冲击力高**：Web 页面可以展示研究图谱、证据矩阵、争议点聚类和空白区域，适合比赛演示和视频呈现。
4. **证据链组织更完整**：每篇论文不只是被推荐，而是被放入问题、方法、数据、结论和局限关系中，便于复核和扩展。
5. **与项目名称契合**：ScholarLoop 的“证据链推荐智能体”定位可以通过图谱闭环表达得更清楚。

### 3.4 主要风险

1. **图谱抽取误差会放大**：方法、数据、结论、局限等字段一旦抽取错误，会影响证据矩阵、图谱关系和研究空白判断。
2. **通用证据结构可能过宽或过窄**：过宽会导致标注和展示复杂，过窄会覆盖不了跨学科案例。
3. **评测难度高**：Precision@10、Recall@20、NDCG 等排序指标容易评测，但“研究空白发现质量”和“图谱关系正确性”需要额外设计人工评价规则。
4. **工程范围容易膨胀**：如果同时追求大规模图谱、漂亮可视化和复杂关系推理，可能超出个人参赛可控范围。
5. **实时演示稳定性要求高**：图谱生成和展示如果不稳定，会影响 Web Demo 的答辩体验。

### 3.5 时间与资源成本

| 维度 | 成本判断 |
|---|---|
| 时间成本 | 高。除基础检索与排序外，还需要定义证据结构、抽取图谱要素、组织图谱展示、设计空白发现评价方式。 |
| 工程成本 | 高。主要成本在证据单元抽取、关系组织、图谱/矩阵可视化、图谱与排序结果联动。 |
| 数据成本 | 高。需要更细粒度地标注或校验方法、数据、结论、局限、争议点和研究空白。 |
| 算力成本 | 低到中。无需昂贵 GPU，但需要较多摘要理解、证据抽取和关系整理调用。 |
| 展示成本 | 中到高。图谱展示能带来冲击力，但也需要控制复杂度，避免页面成为炫技可视化。 |

### 3.6 不确定度

**高。**

路线 B 的创新性更强，但依赖证据结构设计、抽取准确性和图谱展示质量。它更可能形成一等奖级别的差异化，也更可能因范围膨胀影响交付稳定性。

### 3.7 最危险假设

**一套跨案例通用的证据图谱结构能够稳定覆盖碳价预测、AI 科研智能体和跨学科案例。**

如果这个假设不成立，系统会在不同案例之间频繁修改结构，导致评测集、证据矩阵、图谱展示和研究空白判断难以统一，也会削弱项目的通用科研智能体定位。

### 3.8 与冻结成功标准的对应关系

| 冻结成功标准 | 路线 B 的对应方式 |
|---|---|
| 6 个核心功能模块 | 在检索和排序基础上，以证据图谱承接证据矩阵、引用归因、研究空白和后续选题建议。 |
| Web 完整演示与 3 个案例 | Web 可展示复杂问题拆解、论文证据图谱、方法/数据/结论矩阵、争议点和空白区域；案例不绑定碳价预测。 |
| 50 问评测集与基线对比 | 保留排序指标评测，同时增加图谱字段准确率、研究空白人工评价等辅助评价。 |
| 至少 2 个核心指标优于基线 | 可争取人工评价得分、证据归因准确率、NDCG，同时保留 Precision@10 或 Recall@20。 |
| 不允许虚构引用 | 图谱节点必须绑定真实论文来源、作者年份、DOI 或公开来源；未确认关系进入人工核验状态。 |
| 每篇论文 10 项证据字段 | 图谱节点和边天然承载标题、作者年份、来源、推荐理由、支持问题、方法、数据、结论、局限和关联强度。 |
| 显式区分证据状态 | 可通过节点/边状态标记已有证据支持、证据不足、存在争议、需要人工核验。 |
| 创新性 | 强化证据矩阵化组织、研究图谱、研究空白发现和复杂科研问题结构化理解。 |
| 个人参赛硬约束 | 必须控制图谱规模，优先做小而精、可复现、可演示的案例级图谱，不做大规模分布式知识库。 |
| 明确不做事项 | 不做普通论文列表，不做黑箱推荐，不做只追求可视化但缺少证据链的页面。 |

## 4. 路线 C：审稿式多智能体自我修正路线

### 4.1 核心思想

把 ScholarLoop 设计成一个模拟“研究生查文献 + 导师追问 + 审稿人质检”的多角色科研检索流程。系统不是一次性返回结果，而是围绕复杂问题进行多轮计划、检索、筛选、质检、补检和证据归因。

路线 C 的核心是“过程可信”：

1. 查询规划角色负责识别研究对象、方法、数据、指标、争议点和应用场景；
2. 检索角色根据拆解后的子问题进行多轮检索；
3. 筛选角色判断论文与子问题的相关性和证据强度；
4. 质检角色检查虚构引用、来源缺失、证据不足和结论外推；
5. 综合角色形成推荐清单、证据矩阵、研究空白和后续选题建议；
6. 系统保留关键过程日志，使用户能看到为什么需要补检、为什么剔除某些论文、为什么某些结论需要人工核验。

### 4.2 关键假设

1. 多角色分工和自我修正能够显著提升复杂学术查询的召回质量、证据归因准确率和人工评价得分。
2. 大模型在查询拆解、摘要理解、证据组织和质检提示方面足够稳定，且可以通过约束输出、来源绑定和过程记录降低幻觉风险。
3. 多轮检索的额外时间和调用成本在 Web 演示中可以被接受，不会导致系统体验过慢或结果不可复现。
4. 评委会认可“多轮检索与自我修正机制”作为区别于普通 RAG 和普通搜索网页的重要创新点。
5. 通过保存检索结果、过程日志和中间决策，可以满足评测可复查、可复现的要求。

### 4.3 预期收益

1. **智能体特征明显**：相比静态检索排序，路线 C 更符合“智能系统自动完成问题拆解、多轮检索、筛选、归因、排序和空白总结”的闭环叙事。
2. **复杂问题适配性强**：对于含有争议点、条件限制、外生变量失效、严格预测协议等复杂问题，多轮补检和质检更容易体现系统价值。
3. **演示过程有冲击力**：Web 页面可以展示从初始问题到子问题、首轮检索、证据不足提示、补充检索、最终证据矩阵的动态过程。
4. **风险显式化能力强**：可以把“证据不足、存在争议、需要人工核验”放入质检角色的输出中，而不是只在最终答案中附带说明。
5. **有利于消融实验**：可以比较有无查询拆解、有无补检、有无质检、有无综合排序时的指标变化。

### 4.4 主要风险

1. **非确定性较高**：多轮大模型调用可能导致同一问题多次运行结果不一致，影响评测复现和答辩稳定性。
2. **调用成本和延迟较高**：多角色、多轮检索、多次质检会增加响应时间和 API 成本。
3. **容易被认为是“套壳智能体”**：如果没有清晰的评测、基线对比和可解释证据归因，智能体流程本身不足以支撑一等奖标准。
4. **质检不等于真实正确**：自我修正只能降低风险，不能替代真实来源校验和人工抽检。
5. **流程复杂度可能压缩核心实验时间**：如果过度打磨角色协作细节，可能影响 50 问评测集、基线对比和 3 个完整案例的完成质量。

### 4.5 时间与资源成本

| 维度 | 成本判断 |
|---|---|
| 时间成本 | 中到高。需要设计多轮流程、过程记录、质检机制、补检策略、最终结构化输出，并保证演示稳定。 |
| 工程成本 | 中到高。主要成本在流程编排、状态保存、失败回退、过程展示和结果可复现。 |
| 数据成本 | 中。仍需 50 问评测集和人工标注，但图谱级细粒度标注压力低于路线 B。 |
| 算力成本 | 中。无需昂贵 GPU，但多轮大模型调用和多源检索会带来更高 API 调用量。 |
| 展示成本 | 中。过程展示本身有冲击力，但需要把过程压缩成评委易理解的主线。 |

### 4.6 不确定度

**高。**

路线 C 的上限较高，但稳定性、可复现性、延迟和成本都存在明显不确定性。它能否超过基线，取决于多轮自我修正是否真正改善检索推荐和证据归因，而不是只增加流程复杂度。

### 4.7 最危险假设

**多轮自我修正能够在不破坏可复现性和演示稳定性的前提下，显著提升至少 2 个核心评价指标。**

如果这个假设不成立，路线 C 会变成高成本、高延迟、难复现的智能体包装，难以满足评测成功标准和个人参赛可控性要求。

### 4.8 与冻结成功标准的对应关系

| 冻结成功标准 | 路线 C 的对应方式 |
|---|---|
| 6 个核心功能模块 | 多角色流程分别覆盖输入理解、查询拆解、检索、过滤排序、证据归因、空白建议。 |
| Web 完整演示与 3 个案例 | Web 可展示多轮过程日志、补检原因、剔除原因、证据不足提示和最终结构化结果；案例覆盖不同领域。 |
| 50 问评测集与基线对比 | 可把完整多智能体流程作为 ScholarLoop 完整流程，与普通关键词、BM25、Embedding、单轮 LLM 检索对比。 |
| 至少 2 个核心指标优于基线 | 重点争取 Recall@20、人工评价得分、证据归因准确率，也可争取 Precision@10 和 NDCG。 |
| 不允许虚构引用 | 通过质检角色、来源绑定、低置信度提示和过程日志降低虚构引用风险。 |
| 每篇论文 10 项证据字段 | 综合角色汇总每篇论文的标题、作者年份、来源、推荐理由、支持问题、方法、数据、结论、局限、关联强度。 |
| 显式区分证据状态 | 质检角色负责标注已有证据支持、证据不足、存在争议、需要人工核验。 |
| 创新性 | 强化查询拆解、多轮检索、自我修正、证据归因、消融实验和面向研究生科研流程的闭环演示。 |
| 个人参赛硬约束 | 必须限制角色数量、轮次和展示复杂度，不做无限循环式科研代理，不依赖大规模训练。 |
| 明确不做事项 | 不做通用聊天机器人，不做全自动替代科研判断，不做无来源的生成式答案，不做只靠页面过程动画的展示。 |

## 5. 路线差异边界

为避免候选路线只是措辞变化，三条路线的判断标准如下：

| 判断维度 | 路线 A：可解释检索排序 | 路线 B：证据图谱 | 路线 C：多智能体自我修正 |
|---|---|---|---|
| 第一优先级 | 推荐结果可评测、可解释、可复现 | 证据关系结构化与研究空白发现 | 多轮过程、补检和质检能力 |
| 核心资产 | 50 问评测集、评分规则、基线/消融结果 | 证据图谱、矩阵空缺、争议/局限关系 | 角色流程、过程日志、补检/质检记录 |
| 主要创新表达 | 复杂查询拆解 + 综合排序 + 证据矩阵 | 论文—方法—数据—结论—局限图谱 | 审稿式多轮检索与自我修正 |
| 最大收益 | 稳定达成验收指标 | 研究空白发现更有冲击力 | 智能体闭环展示更明显 |
| 最大风险 | 创新感不足 | 范围膨胀和抽取误差 | 非确定性、延迟和复现难度 |
| 个人可控性 | 较强 | 中等偏弱 | 中等 |

## 6. 共通不可跨越边界

无论后续选择哪条路线，都不得改变以下边界：

1. 不把项目改成通用聊天机器人。
2. 不把项目降级成普通关键词论文搜索网站。
3. 不只返回论文标题列表，必须输出结构化科研证据链。
4. 不取消 50 问评测集、基线对比和消融实验。
5. 不依赖大规模 GPU 训练或不可公开数据。
6. 不使用未经授权的付费全文数据。
7. 不伪造 DOI、作者、期刊、年份或实验结果。
8. 不宣称系统可以完全替代人工科研判断。
9. 不把碳价预测作为唯一核心场景。
10. 不把页面视觉效果置于问题拆解、检索推荐、证据归因和研究空白发现之上。

## 7. 本文件结论

本文件仅完成候选路线发散，不选择最终路线。

当前形成 3 条真正不同的候选路线：

1. **路线 A：评测驱动的可解释检索排序路线**，以稳定、可解释、可复现和指标优势为中心。
2. **路线 B：证据图谱驱动的研究空白发现路线**，以结构化证据关系、研究图谱和空白发现为中心。
3. **路线 C：审稿式多智能体自我修正路线**，以多轮检索、质检、补检和过程可信为中心。

三条路线均服务于冻结问题定义，均保留 Web 展示形态、3 个完整案例、50 问评测集、基线对比、证据矩阵、研究空白分析和不生成虚假引用等硬约束。
