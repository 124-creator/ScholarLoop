## 4. 调研结论（须由执行方现场核验，不得当作既定事实）
- **候选数据源**（点名，许可/字段/频率自行核验）：
  - OpenAlex（REST API，通常免 key、建议带 mailto 进 polite pool）；
  - Semantic Scholar Academic Graph API（可申请 key，有速率限制）；
  - Crossref（DOI 解析与元数据校验，建议带 mailto）；
  - arXiv（开放全文片段）。
- **公开基准金标准**（优先级）：
  1. LitSearch（princeton-nlp/LitSearch；EMNLP 2024）——自带 query + 相关性标注 + 语料；
  2. PaSa / RealScholarQuery（github.com/bytedance/pasa）；
  3. AstaBench / PaperFindingBench（github.com/allenai/asta-bench）。
  > 优先选 **ground truth 可直接获取且许可允许**的那个作为 S-X1 金标准；选定理由写入结论。
- **对本计划的影响**：用公开基准官方标注作金标准，直接化解 X1「自建自评」利益冲突；语料由基准给定，化解 X2「苹果比橘子」。

## 5. skills 选型与执行简报（已转译为工具无关步骤）

**选用能力（按本项目语境转译，不要求执行方自行读 skill）**：
- 学术 API 检索与取数（对应 arxiv / semantic-scholar / research-lit 类能力）；
- 外部文档/许可核验（对应 docs / external-context 类能力）；
- 结果统计与对比表（对应 analyze-results 类能力）。

**未选**：run-experiment（GPU 部署）/ paper-* / 图谱类——本阶段是小样本本地探针，不需要远程算力或论文产出。

**转译后的硬步骤（H=硬要求，S=建议）**：
- H1：每个外部来源在使用前，先抓取其**官方文档/许可页**，记录许可、可用字段、速率/配额，存为证据文件。
- H2：所有外部 API 原始响应（JSON）必须落盘到 `spike/raw/`，可复查可复现。
- H3：LLM 调用一律 **temperature=0**，可固定处固定随机种子；记录模型名与版本。
- H4：指标用标准定义（Precision@K、Recall@K、F1）；评测脚本必须可重复运行且两次结果一致。
- H5：**0 容忍虚构**——任何无法在来源解析到的论文/DOI/作者，不得写入证据字段，须标「证据不足/需人工核验」。
- S1：优先用轻量开源组件（如 BM25 用 `rank_bm25`、嵌入用小型开源 sentence-transformers）控制成本。
- S2：能本地缓存的检索结果就缓存，降低重复调用。

### 16.2 T2b · LLM 基线 + A-min（不确定度：中高；**待端点 env 配好**）
- 前置：T2b 端点经环境变量注入、**绝不写入仓库**（密钥值仅在本地 env；本文件只记非密的 base_url/model）：

  - **备用 · 火山方舟 Ark**：`LLM_BASE_URL=https://ark.cn-beijing.volces.com/api/coding`（用户提供，须预检确认；标准 OpenAI 兼容路径常为 `/api/v3`）、`LLM_API_KEY`（env，`ark-` 开头）、`LLM_MODEL`=火山推理接入点 ID（如 `ep-xxxx`，**若落到此路径且用户未提供 model 须停机索取**）。
