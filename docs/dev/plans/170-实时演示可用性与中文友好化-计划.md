## 1. 模块目标

- **本模块解决什么**：①让实时"问任意问题"**稳定快速可用**（缓存提速 + 放宽/重构预算 + 预热），评委每次点都能在预算内看到真实排序推荐；②把实时结果卡 + 点即核验面板**中文友好化**（保留英文 SCI 标题/摘要，旁边加中文标签/解释），让评委读懂"零幻觉"卖点。
- **对冻结成功标准的贡献**：S4 落地（实时自主搜索从"时好时坏"变"稳定可用"）+ 专家评审对外呈现质量。
- **诚实性要求**：实时非 verified、真失败优雅回退不编造；提速只缓存不改结果；中文化只改显示不改 verified 语义/忠实逻辑。

## 2. 范围

- **本轮必须做**：
  1. **T1 · 实时缓存提速（realtime.py·非承重路径）**：把 `BM25Retriever`/`DenseV2Retriever`/`CrossEncoderReranker`/语料 **跨调用缓存复用**（首次构建、后续秒级），**断言缓存后结果与未缓存逐条相同**（相同 query → 相同 corpusid/score/排序）。
  2. **T2 · 预算放宽 + 不丢已算结果 + 诚实成本**：放宽/重构 `timeout_s`（如 45→180）使**已算完的结果不被丢弃**；若真超时/失败仍优雅回退，且**显示真实耗时/调用数（不再硬编码 cost=0）**；可选服务启动**预热**一次。
  3. **T3 · 实时结果卡中文友好化（design.py + i18n.py + studio.py 传 i18n 键）**：`Question was decomposed into`→"已将问题拆解为"；`#N · paper X · score Y`→"第N名 · 论文X · 综合得分Y"；`Ranking signal: bm25=1.000*0.10...`→"排序依据"中文标签（原始权重串折叠进"技术细节"或简化中文化）；`Authors/year/DOI stay for manual verification`→中文"作者/年份/DOI 需人工核验"；**保留英文论文标题/摘要**。
  4. **T4 · 点即核验面板中文化（design.py·显示层）**：`Manual verification required. source_field missing or not locally verifiable. If the source sentence cannot be matched exactly, ScholarLoop does not highlight or guess`→中文解释（把"零幻觉:匹配不到原句绝不高亮绝不猜测"讲清楚）；**映射在 JS 显示层完成，不改 `source_text.py`/`interactive.py` 忠实逻辑**。
  5. **T5 · 示例查询稳定可检索（i18n.py EXAMPLE_QUESTIONS）**：校验每个示例 chip 在提速后**稳定返回 ≥5 条真实结果**，替换任何不稳定项为 LitSearch 域内稳定项；可加简短中文提示。
  6. **承重复验 + 合规 / 可复现 / 复盘**。
- **明确不做**：改 A-v2 检索逻辑/融合权重/缓存键/任何 verified 数值；改点即核验忠实逻辑（`source_text`/`interactive`）；翻译 verified 数值/证据原文/论文标题摘要；为可用而编造实时结果；引重前端框架/外链字体；改 M040/M060 冻结评测产物。**凡触及 verified 数据或忠实逻辑一律停机上报。**
- **允许修改 / 新建的文件**：
  - 实时提速：`src/scholarloop/demo/realtime.py`（缓存复用 + 预算 + 诚实成本，**不改检索逻辑/权重/结果**）。
  - 中文化：`src/scholarloop/demo/{design.py, i18n.py}`（结果卡/核验面板 JS 显示层 + 中文字典 + 示例查询）、`src/scholarloop/demo/studio.py`（**仅扩充传给 JS 的 `window.SL_I18N` 键，不改 verified 渲染语义**）。
  - 新建测试：`tests/test_m170_*.py`（**不改** `test_m010..m160`）。
  - 新建产物：`reports/m170/**`（缓存等价证据、实时示例稳定证据、中文化证据、承重复验、扫描、validation_summary）。
  - **不得**改动：FROZEN 件、`spike/**`、`reports/m010..m160/**`（数据只读，m040/m060 评测产物/缓存键冻结）、`demo/{source_text.py,interactive.py,assemble.py,enrich_view.py}` 既有逻辑、`retrieval/`/`rank/` 检索类逻辑/权重、各模块验收判据、任何 verified 数值。

## 3. 现状与证据

- **实时误杀根因**：`realtime.py:55` `timeout_s=45.0`；`:119-120` `if elapsed > timeout_s: return unavailable("...超过预算阈值...")`（丢弃已算结果）；`:75` 每次 `BM25Retriever(docs)` 重建 6.4 万篇索引；`:76`/`:95` 每次重载 Dense/CrossEncoder；`unavailable()`（`:29-39`）硬编码 cost=0。总指挥实测：流水线真返 10 条、~30s 压线、时好时坏。
- **英文/黑话源**：`design.py:133`（decomposed into）、`:134`（paper/score/Ranking signal/Authors stay for manual verification）、`:147`（Manual verification required/does not highlight or guess）；状态码 `source_text.py:62`（manual_review_reason 英文，**显示层中文化、不改后端**）。
- **可复用资产**：M140 i18n 框架（zh/en 字典 + tr 取词）；缓存复用为标准工程提速；点即核验忠实逻辑 M120 已 verified（只读复用）。

## 4. 调研结论

- **来源**：总指挥读码定位（realtime.py / design.py / source_text.py）+ 实时端到端实测（真返 10 条、~30s 压线、45s 阈值误杀）。
- **可直接落地的结论**：① 提速纯缓存复用、结果不变、可断言等价；② 预算放宽/不丢结果为非承重演示参数；③ 中文化均在 JS 显示层 + i18n、不碰忠实逻辑；④ 示例查询提速后稳定。
- **对本计划的影响**：承重 = 实时稳定可用且结果等价 + 冻结评测零变化 + 点即核验忠实不破 + 中文化不改 verified 语义。
