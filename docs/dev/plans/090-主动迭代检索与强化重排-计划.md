## 1. 模块目标

- **本模块解决什么**：把 A 主干从**单轮检索排序**升级为 **A-v3 主动迭代检索 + 强化重排**——①根据首轮已检出的相关论文**迭代重构子查询、再检索、合并候选**（官方 §3.1.2）；②对扩展候选做**强化重排**抬 P@10；③（条件触发）引文扩展。目标是**抬高绝对 F1**（42% 主轴），同时补齐官方核心功能缺口。
- **对冻结成功标准的贡献**：
  - **S1 / F1（70%·主轴）**：A-v3 端到端 F1 **配对显著优于 A-v2 冻结基线**，在留出/test 集成立（隐藏集 30% 代理）。
  - **S2 落地（§3.1.2）**：实现「迭代式检索：根据已找到论文动态调整策略」，补官方核心功能。
  - **S2 效率（20%）**：迭代/重排引入 LLM 成本——**诚实记账**（调用/token/延时），报告 A-v3 vs A-v2 成本增量，确保增益对得起成本。
- **诚实性要求**：A-v3 若**不显著优于 A-v2**、或增益只在 train 集成立（过拟合）、或只能靠 **gold 泄漏 / 候选伪造** 取得，如实停机上报，**绝不**伪造提升。

## 2. 范围

- **本轮必须做**：
  1. **T0 · 可行性桩（gate）**：小样本证明「迭代检索 + 强化重排」在**不泄漏 gold** 的前提下能把 F1 抬向 gold（相对 A-v2 冻结正向且方向稳定）；并评估**引文扩展离线可行性**（LitSearch `full_paper` 参考文献解析 / 连接器联网，二者成本与可达性）。桩不正向或只能靠 gold 泄漏 → 停机或砍引文扩展支。
  2. **迭代检索（多轮，gold-blind）**：`retrieval/iterative.py` + `agent/`——首轮 A-v2 检索 → LLM 读 top-k **标题/摘要（非 gold）** 生成精炼子查询 → 再检索 → 合并去重扩展候选池；轮数/预算有界、确定性（temperature=0/seed）、原始 LLM 落盘。
  3. **强化重排**：`rank/rerank_v2.py`——对扩展候选做更强重排（listwise LLM 重排 / 更大 cross-encoder），抬 P@10；同候选池内排序、H5 不越池。
  4. **（条件触发）引文扩展**：仅当 T0 桩证明离线可行（不联网伪造、cited 论文须在 gold 语料内）→ 沿引文邻域拉新候选；否则**如实砍掉**并在复盘记录。
  5. **端到端评测 + 显著性 + 成本**：`eval/run_full_v3.py`——A-v3 vs A-v2 冻结 vs 基线在**同 gold** 上端到端算 P@10/R@20/F1/NDCG，**沿用 M040 的 train/holdout/test split**（调参 train-only、留出/test 只验证），配对 bootstrap≥10000/置换；效率记账。产出 `reports/m090/{results.json,significance.json,评测报告.md}`。
  6. **合规 / 可复现 / 复盘**。
- **明确不做**：改 M010–M080 任何产物/源码行为/验收判据/A-v2 配置（**A-v2 冻结为对照基线**）；**gold 泄漏进检索/重排**（红线）；**伪造候选**（只从离线语料取，引文扩展的 cited 论文须真实在 gold 语料内）；为引文扩展**强行联网抓取**冒充离线（条件触发须显式标注口径）；重训嵌入/重排基模型（用现成开源权重）；挑样本/调判据制造通过。

## 3. 现状与证据

- **F1 瓶颈实测**：A-v2 LitSearch R@20=**0.7564**、P@10=**0.0727**、F1=**0.1312**；RealScholarQuery F1=**0.1972**。**召回已高、精度卡死** → 抬 F1 = 抬 P@10（强化重排）+ 回收 24% 未进 top-20 的 gold（迭代/引文扩展）。
- **管线现状**：`retrieval/{bm25,dense,dense_v2}` + `rank/{fusion,fusion_v2,rerank}` + `query/decompose`——**单轮，无迭代/无引文扩展**。
- **引文边可行性（已核实）**：RealScholarQuery 语料仅 `corpusid/arxiv_id/title/abstract/title_key/source`、PaSa `id2paper` 仅 id→title——**均无显式引文边**；LitSearch parquet 有 `full_paper`（全文，可解析参考文献但重/噪/单基准）。**连接器** `openalex.py` 有 `cited_by_count`、S2 连接器存在——联网可得引文但引成本+可达性。→ **引文扩展是条件触发件，非承重依赖**。
- **可复用资产**：M040 的 split（`int(sha256(qid)[:8],16)%10`）、同候选池/H5/bootstrap、bge-small 嵌入、cross-encoder；M010 的 LLM 客户端 + 原始落盘 + temperature=0/seed 范式；M050 连接器（引文扩展条件触发时）。

## 4. 调研结论

- **来源**：官方 §3.1.2 + 题目「引文网络探索」、参考系统（PaSa Crawler+Selector 多轮、SPAR RefChain 查询演化、Ai2 多索引+引文追踪）、015 F1 主轴、M040 防过拟合协议。
- **可直接落地的结论**：
  1. **F1 是精度瓶颈** → 强化重排（listwise LLM / 更大 cross-encoder）是直接杠杆。
  2. **迭代检索**（多轮重构+再检索）是官方点名、冠军共用的核心手段，且**离线纯语料可行、gold-blind 可守**。
  3. **引文扩展**受限于离线无引文边 → 条件触发，不绑承重。
  4. 迭代/重排引入成本 → **效率 20% 需诚实记账**，增益须对得起成本。
- **对本计划的影响**：承重锁定「A-v3 端到端 F1 配对显著 > A-v2 冻结，留出/test 成立」；迭代/重排为主路径，引文扩展条件触发。

## 5. skills 选型与执行简报

- **选择的 skills**：无（迭代编排 + 重排 + 端到端评测为确定性工程；LLM 重构/重排受协议约束）。
- **未选其它 skills 的理由**：承重在端到端 F1 显著性协议，靠留出集防过拟合，不靠编排花活。
- **转译后的普通执行步骤（硬要求标【硬】 / 建议标【议】）**：
  1. 【硬】迭代检索 / 重排 **绝不读 gold**（gold-blind）；候选只从离线语料取，不伪造。
  2. 【硬】**T0 桩先过**：不泄漏 gold 下相对 A-v2 正向；否则停机/砍引文支。
  3. 【硬】承重 = A-v3 端到端 F1 **配对显著 > A-v2 冻结**，且**留出/test 成立**（防过拟合）；调参 train-only。
  4. 【硬】成本诚实记账（调用/token/延时），报 A-v3 vs A-v2 增量；temperature=0/seed、原始 LLM 落盘、可复现。
  5. 【硬】同候选池内排序公平、H5 不越池；上游 M010–M080 + A-v2 配置冻结只读。
  6. 【议】轮数/候选预算/重排实现（listwise vs cross-encoder）/引文扩展是否启用，在 §10 预算内定并落盘依据。

## 10. 执行方有界决策预算

### 可自行决定
- 迭代轮数与候选预算；子查询重构 prompt；重排实现（listwise LLM / 更大 cross-encoder）；引文扩展是否启用与口径（须满足不伪造/不泄漏）；缓存键；`agent/` 文件组织与局部命名。

## 11. 风险、边界与回滚

- **核心风险（承重）**：迭代+重排可能**不显著优于 A-v2**（单轮已接近该语料上限）。**真实信号** → 停机如实上报，由人类决定（接受「A-v2 已是该语料强基线」/ 换重排模型 / 调轮数），**绝不**调判据或挑样本。
- **gold 泄漏红线**：检索/重排只用 query + 语料，**绝不**读 gold；T0/T4 须有 gold-blind 校验。
- **过拟合红线**：调参 train-only，留出/test 只验证；增益须跨 split 成立。
- **成本边界**：诚实记账，增益对不起成本则如实标注（效率 20% 维度）。
- **引文扩展边界**：离线无引文边 → 条件触发、不绑承重、不联网伪造。
- **回滚**：A-v3 为新增并行路径，A-v2 + 上游全部冻结只读；失败仅废弃 `src/scholarloop/{retrieval/iterative.py,agent,rank/rerank_v2.py,eval/run_full_v3.py}` 与 `reports/m090/**`，A-v2 对外 F1 声称不受影响。
