## 1. 模块目标

- **本模块解决什么**：M040 证明 A-v2 在 **LitSearch（我们调参/评测所在基准）** 上显著优于基线。但泛化分（10%）与隐藏集（30%）要的是「换一个基准/领域也成立」。本模块把 A-v2 的**冻结配置**搬到**第二个公开基准**（自带官方金标准），证明其 F1 优势**不是 LitSearch 专属**。
- **对冻结成功标准的贡献（映射 015 S1–S5）**：
  - **S4 泛化（10%）**：本模块的**主交付**——跨基准/跨域 F1 优势 + 隐藏集稳健性代理证据。
  - **S1（金标准来源）**：用第二个公开基准官方 gold（RealScholarQuery / AstaBench-PaperFinding 等），**不自标注** → 化解 X1。
  - **X2（同候选池）/H5（0 虚构）/可复现**：继承 M040 协议不放宽。
  - **S2/F1 本身不在本模块**：不改 A-v2 配置或 LitSearch 结果（M040 冻结）。
- **诚实性要求**：A-v2 用**冻结配置**（不在新基准上重调）跑，才算真泛化；若需重调才赢，须诚实并列报告。

## 2. 范围

- **本轮必须做**：
  1. **选定并获取第二个公开基准**：优先 **RealScholarQuery**（`github.com/bytedance/pasa`），备 **AstaBench / PaperFindingBench**（`github.com/allenai/asta-bench`）；自带官方相关性 gold。落数据源/许可/gold 协议。
  2. **基准适配**：复用 M010 `CorpusRepository` 接口，实现第二基准的语料 + 查询 + gold 适配器（离线）。
  3. **冻结配置迁移评测**：用 **A-v2 的 M040 冻结配置**（bge-small 嵌入 + 融合权重 + 可选重排，**不重调**）+ 基线（关键词/BM25/Embedding/单轮LLM）在第二基准上**同候选池**算 P@10/R@20/F1/NDCG。
  4. **显著性**：A-v2 vs BM25 在第二基准上配对 bootstrap(≥10000)/置换；记录效率。
  5. **（诚实并列）**：若冻结配置在第二基准上不显著优于 BM25，**额外**报告"在第二基准 train 子集重调"的结果，**明确标注两者差异**，不以重调结果冒充零样本泛化。
  6. **可复现 + 合规 + 复盘**。
- **明确不做**：改 M010–M050 产物/判据/A-v2 配置（全部冻结只读）；自建/自标注 gold（X1 红线）；重训/微调模型；GPU 大训练；付费/受限数据；Web/图谱/研究空白（M070/M080）；任何虚构。
- **允许修改 / 新建的文件**：
  - 新建：`src/scholarloop/benchmarks/**`（第二基准适配器）、必要的评测入口 `src/scholarloop/eval/run_benchmark2.py`。
  - 新建测试：`tests/**`（`test_m060_*.py`，**不改** `test_m010..m050`）。
  - 新建产物：`reports/m060/**`（含 `cache/`、`raw/`）。
  - **不得**改动：FROZEN 件、`spike/**`、`reports/m010|m020|m030|m040|m050/**`、M010–M050 既有源码行为（**A-v2 配置冻结复用，不改**）、各模块验收判据。

## 4. 调研结论

- **来源**：`015` S1（公开金标准清单）、`02-评审分级.md` X1/X2、RealScholarQuery/PaSa 与 AstaBench 公开仓库与论文、M040 verified 产物。
- **可直接落地的结论**：
  1. **第二公开基准 + 官方 gold** 是 X1-安全的泛化证据（不自标注）。
  2. **冻结配置零样本迁移**是最强泛化主张；重调结果只能作并列、明确标注。
  3. **同候选池 + 显著性 + H5** 协议直接移植 M040，保证可比与零虚构。
- **对本计划的影响**：T0（基准获取）与 T3（冻结迁移显著性）是 make-or-break；获取不到/不显著 → 停机上报，**不得**自标注 gold 或重调冒充零样本。
