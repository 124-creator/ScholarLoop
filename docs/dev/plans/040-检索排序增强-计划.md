## 1. 模块目标

- **本模块解决什么**：官方 70% 压在 F1（公开 30% + 隐藏 30%）。M010 已证明 A-v1 显著>BM25，但**绝对 F1=0.1128 仍低**。本模块用**更强但开源、可离线、确定性**的检索排序栈，把 A 的 F1 **稳健**抬高，并以**留出集/交叉验证**证明增益不是过拟合 LitSearch 公开集（隐藏集的代理证据）。
- **对冻结成功标准的贡献（映射 015 S1–S5）**：
  - **S1（F1·70%）**：本模块的**主交付**——A-v2 在全量 LitSearch 同候选池下，F1 **配对显著优于 BM25**，且**优于 A-v1**，并在留出集上成立（防过拟合）。
  - **S2（效率·20%）**：嵌入升级 + 重排会增 Token/算力，须全程记账并控制在 `015` S2 预算内；超预算即停机。
  - **S4（落地/泛化）**：增益来自**原则性方法**（更强嵌入 + cross-encoder 重排），非调参刷公开集 → 更可能泛化、经得起隐藏集与复现质询。
  - **S5（合规）**：H5「0 虚构」、同候选池、确定性、原始落盘**全部继承 M010，不放宽**。

## 2. 范围

- **本轮必须做**：
  1. **神经嵌入升级**：`all-MiniLM-L6-v2`（地板）→ 更强开源模型（`bge-base-en-v1.5` / `e5-base-v2` 等），确定性编码，模型名+版本入证据；语料编码缓存。
  2. **Cross-encoder 重排**：对候选池 Top-N 用开源 cross-encoder（如 `bge-reranker-base` / `ms-marco-MiniLM`）精排；消融其净 F1 贡献（开/关对比）。
  3. **启用子查询信号**：M010 因 smoke 小样本把 `sub_bm25/sub_dense` 权重置 0；在**全量**上重估，净正才纳入融合。
  4. **融合 v2 重调（防过拟合）**：在**留出集 / k-折**上调融合权重与 Top-K，不在整 LitSearch 公开集上调；落盘调参协议与最终配置。
  5. **全量 LitSearch 重测**：≥5 系统（关键词/BM25/神经Embedding/单轮LLM/**A-v2**，并保留 **A-v1** 作对照）同候选池算 P@10/R@20/F1/**NDCG**；对 **A-v2 vs BM25** 与 **A-v2 vs A-v1** 各做配对 bootstrap/置换检验；效率记账。
  6. **防过拟合证据**：留出/交叉验证证明增益不是 LitSearch 公开集调参所得（隐藏集稳健性代理）。
- **明确不做**：改 M010/M020/M030 任何产物、代码行为或验收判据（**A-v1 冻结**）；过拟合公开集；GPU 大训练 / 模型微调 / >2GB 重依赖受阻仍硬上；在线付费 / 受限数据；B-lite/C-lite/Web/图谱；任何虚构。
- **允许修改 / 新建的文件**：
  - 新建增强检索/排序（**新路径，不回改 M010**）：`src/scholarloop/retrieval/dense_v2.py`、`src/scholarloop/rank/rerank.py`、`src/scholarloop/rank/fusion_v2.py`、`src/scholarloop/eval/run_full_v2.py`（或参数化扩展，保留 v1 行为）。
  - 新建测试：`tests/**`（新增 `test_m040_*.py`，**不改** `test_m010/m020/m030`）。
  - 新建产物：`reports/m040/**`（含 `raw/`、`cache/`）。
  - **不得**改动：任何 FROZEN 件、`spike/**`、`reports/m010|m020|m030/**`、M010/M020/M030 既有源码行为、`010` 验收判据、`040 结论`。

## 3. 现状与证据

- **已存在可复用（M010，verified）**：`spike/eval/run_spikes.py` 与 `src/scholarloop/eval/run_full.py` 的同候选池协议、`metric_for_ranking`、H5 剔除、LLM 封装、bootstrap；语料 64,183 篇 + LitSearch 金标准；A-v1 结果 `reports/m010/results.json`（**冻结对照**）。
- **已确认事实（审 M010 代码所得，本模块的提升空间）**：
  - 嵌入是地板模型 `all-MiniLM-L6-v2`；
  - **无 cross-encoder 重排**（当前只有分数融合）；
  - 融合权重 `bm25=0.40, dense=0.60, sub_bm25=0, sub_dense=0`（子查询信号被关）；
  - A-v1 F1=0.1128、BM25=0.0964、ΔF1=0.0164（CI[0.0112,0.0217]）。
- **仍未知但可在执行中确认**：升级后能否在全量上**显著优于 A-v1**（核心风险，§11）；最佳嵌入模型/重排器/融合;CPU 下全量编码+重排的时延是否在预算内（可能触发停机）。

## 4. 调研结论

- **来源**：M010 verified 产物与代码、`015` S1/S2、检索排序常识（更强双编码器 + cross-encoder 重排是检索任务最稳的涨点路径）、LitSearch（arXiv:2407.18940）。
- **可直接落地的结论**：
  1. **更强嵌入 + cross-encoder 重排**是原则性、可复现、易泛化的提升，优先于"调融合权重刷公开集"。
  2. **防过拟合是隐藏集（30%）的关键**：必须在留出/交叉验证上确认增益，落盘 train/test 划分协议。
  3. **保留 A-v1 冻结对照**：A-v2 必须证明优于 A-v1，否则更复杂的管线无意义 → 停机上报。
- **对本计划的影响**：T5（全量重测 + 双显著性 + 防过拟合）是 make-or-break；不优于 A-v1 / 仅公开集过拟合 → 停机，**不得**靠调参制造表面提升。

## 5. skills 选型与执行简报

- **选择的 skills**：无（检索排序增强为确定性工程任务，M010 已给可移植骨架）。
- **未选其它 skills 的理由**：引入额外编排无收益；关键是**更强模型 + 严格留出验证**，靠协议保证而非复杂 agent。
- **转译后的普通执行步骤（硬要求标【硬】 / 建议标【议】）**：
  1. 【硬】嵌入/重排模型开源、可离线、确定性；模型名+版本入证据；编码缓存可复现。
  2. 【硬】同候选池协议、H5、temperature=0/seed、原始落盘**继承 M010 不放宽**。
  3. 【硬】融合/Top-K 调参只在**留出/交叉验证**上做；A-v1 冻结对照同池重测。
  4. 【硬】A-v2 须配对显著优于 BM25 **且**优于 A-v1，并在留出集成立；效率记账在 S2 预算内。
  5. 【议】具体嵌入模型、重排器、Top-N/K、融合权重、向量索引实现，在 §10 预算内决定并落盘依据。

- [ ] **T1（不确定度：中）神经嵌入升级**：`retrieval/dense_v2.py` 用更强开源句向量编码语料+查询、余弦召回；确定性、缓存、模型版本入证据。**【停机】** 需 GPU 或 >2GB 下载受阻 → 上报。
- [ ] **T2（不确定度：中）Cross-encoder 重排**：`rank/rerank.py` 对候选池 Top-N 精排；开/关消融。完成判据：重排确定性、净贡献入报告。

## 11. 风险、边界与回滚

- **核心风险（T5）**：更强模型未必在全量上显著超 A-v1，或增益只在公开集（隐藏集会塌）。缓解 = 原则性方法 + 留出/交叉验证 + A-v1 冻结对照 + 双显著性。**若失败**——是真实信号，停机上报，由总指挥/人类决定（换模型/换重排/接受 A-v1 为最终），**绝不**伪造提升。
- **效率边界**：嵌入+重排增成本，须 S2 预算内；CPU 全量编码/重排时延受阻 → 停机不硬上 GPU。
- **回滚**：A-v2 为新路径，A-v1（M010）冻结不动；失败仅废弃 `src/scholarloop/{retrieval/dense_v2,rank/rerank,rank/fusion_v2,eval/run_full_v2}` 与 `reports/m040/**`，不污染已 verified 证据。
