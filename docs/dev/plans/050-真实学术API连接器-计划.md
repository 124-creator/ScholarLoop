## 1. 模块目标

- **本模块解决什么**：M020 诚实地把离线语料缺失的官方字段（作者/年份/来源/DOI）标为「需人工核验」并指向连接器接口。本模块**落地这些接口**——用免费开放的学术 API（OpenAlex 主、Semantic Scholar/Crossref/arXiv 备）把这些字段**带出处地解析为真实值**，把"接口桩"变成"真对接真实学术 API"。
- **对冻结成功标准的贡献（映射 015 S1–S5）**：
  - **S4 落地（15%）**：本模块的**主交付**——真正"对接真实学术 API"，证据链字段从"需人工核验"升为"已有外部来源支持（带 source+id+时间戳）"。
  - **S5 合规（承重）**：解析值必须**与外部记录一致并带出处**；标题匹配置信不足则**保留「需人工核验」，绝不伪造**作者/年份/DOI/期刊。
  - **X3（评审阻断集）**：数据源**点名** + 许可 / 字段 / 频率核验落盘；限速、礼貌 UA、只用开放免费 API。
  - **S1/S2 不在本模块**：不改 F1（A-v2 冻结）；联网解析为离线增强层，不进每查询推理主路径的 S2 计费（如进 Demo 实时解析须限额，见 M080）。
- **为下游铺路**：真实元数据 + 真实语料检索能力是 **M060 真泛化**（跨域、隐藏集代理）的前提。

## 2. 范围

- **本轮必须做**：
  1. **连接器实现**（落地 M010 接口签名）：**OpenAlex**（主，全开放、免 key、字段全）+ **Crossref**（DOI/期刊）+ **Semantic Scholar** / **arXiv**（备）。
  2. **带出处的元数据解析**：给定论文（标题 / corpusid），解析 `authors / year / venue / doi / oa_status / citation_count`，每个值带 `source + external_id + fetched_at + license`。
  3. **补 B-lite 缺口（核心价值）**：`resolver` 读 M020 evidence 中「需人工核验」的作者/年份/DOI，用 OpenAlex 按**标题强匹配**解析；**匹配置信不足 → 保留「需人工核验」**，不强填。产出**增强证据**到 `reports/m050/enriched/`（**新层，不改 M020**）。
  4. **缓存 + 可复现**：所有响应落缓存（`reports/m050/cache/`），记 source/license/timestamp；离线重放确定性；明确标注"在线再取可能随时间变化"。
  5. **X3 合规**：数据源点名 + 许可/字段/频率落 `reports/m050/data-sources.md`；限速、礼貌 User-Agent；只用开放免费 API。
  6. **有界校验**：抽样证明解析值与外部记录一致、带出处、0 伪造；未验证标题匹配仍「需人工核验」。
- **明确不做**：改 M010/M020/M030/M040 任何产物或判据（**全部冻结只读**）；付费/受限 API；爬全文/绕墙；把在线解析塞进 A-v2 的 F1 主路径（不动 F1）；大图谱/研究空白（M070）；Web 改造（M080）；伪造任何字段。
- **允许修改 / 新建的文件**：
  - 新建：`src/scholarloop/connectors/**`（`base.py`/`openalex.py`/`crossref.py`/`semantic_scholar.py`/`arxiv.py`/`cache.py`/`resolver.py`）。
  - 新建测试：`tests/**`（`test_m050_*.py`，含**录制夹具**离线单测；**不改** `test_m010/m020/m030/m040`）。
  - 新建产物：`reports/m050/**`（`cache/`、`enriched/`、`data-sources.md`、`verification.json`）。
  - **不得**改动：任何 FROZEN 件、`spike/**`、`reports/m010|m020|m030|m040/**`、既有源码行为、各模块验收判据、`040 结论`。

## 4. 调研结论

- **来源**：`00-问题描述.md`（证据字段 + 落地"对接真实学术 API"）、`02-评审分级.md` X3（数据源/许可/字段/频率）、`015` S4、OpenAlex/Crossref/Semantic Scholar/arXiv 公开文档。
- **可直接落地的结论**：
  1. **OpenAlex 为主**：全开放、免 key、含 authorships/publication_year/host_venue/doi/oa/cited_by_count，单源即可覆盖大部分官方字段。
  2. **零伪造的可操作判据**：解析值必须能在外部记录中定位（带 `source+external_id`），且标题匹配过阈值；否则保留「需人工核验」。把 S5 在外部数据上做成可机检（类比 M020 的 char_span）。
  3. **可复现靠缓存快照**：在线数据会变，缓存 + 时间戳 + "可能变化"声明是诚实且可复现的做法。
- **对本计划的影响**：联网可用性（§11）与标题匹配置信是关键；匹配不过阈值**宁可保留「需人工核验」也不强填**。

## 5. skills 选型与执行简报

- **选择的 skills**：无（HTTP 连接器 + 解析 + 校验为确定性工程任务）。
- **未选其它 skills 的理由**：标准 REST 调用 + 缓存，无需额外编排。
- **转译后的普通执行步骤（硬要求标【硬】 / 建议标【议】）**：
  1. 【硬】只用开放免费 API；限速 + 礼貌 UA（含联系方式占位）；如需 key 仅经 `secrets/`，不写仓库。
  2. 【硬】每个解析值带 `source + external_id + fetched_at + license`；标题匹配过阈值才落值，否则保留「需人工核验」。
  3. 【硬】所有响应落缓存可离线重放；不改 M020 原件，增强写新层 `reports/m050/enriched/`。
  4. 【硬】数据源点名 + 许可/字段/频率写 `data-sources.md`（X3）。
  5. 【议】具体阈值、备用源启用、字段映射、缓存键，在 §10 预算内决定并落盘依据。

## 11. 风险、边界与回滚

- **联网可用性（首个联网模块）**：Codex 进程可能无外网。缓解 = T1 先试真实调用；不可联网则交付连接器代码 + 录制夹具 + 离线单测，并停机上报"真实可用"判据待联网环境，**不**伪造在线结果。
- **零伪造风险**：标题匹配可能错配 → 错误作者/DOI。缓解 = 强匹配阈值 + 出处可定位 + 不过阈值保留「需人工核验」；校验桩机检。
- **可复现 vs 在线变化**：在线数据随时间变。缓解 = 缓存快照 + 时间戳 + 明确声明；评测以缓存为准。
- **X3 许可**：只用开放免费源并落盘许可；不碰付费/受限/爬墙。
- **回滚**：本模块为新增层，上游全部冻结只读；失败仅废弃 `src/scholarloop/connectors/**` 与 `reports/m050/**`，不污染已 verified 证据。
