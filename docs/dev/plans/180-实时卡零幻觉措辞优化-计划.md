> **动因（人类实测旗舰 demo 后指出）**：实时结果卡的「作者/年份/DOI 需人工核验（除非已有离线核验缓存）。」措辞**像在给评委派作业**，把我们最强的"零幻觉/绝不臆造"卖点说成了"半成品/缺字段"。**本质澄清**：实时对新问题不现场调连接器（为快+离线），这三字段语料里本就没有 → 系统**诚实标注待核验、绝不编造假作者/假 DOI**（这正是赛题反对的幻觉）；30 个基准查询已通过 OpenAlex 核验并显示真实值。**目标 = 把这句改成"讲价值"的说法，让评委一眼读出"这是零幻觉的严谨，不是缺陷"。**
> **诚实红线**：只改**显示措辞**，**不改任何 verified 数值/证据/论文标题摘要/检索逻辑/点即核验忠实逻辑**；实时仍诚实（绝不臆造作者/DOI，语义不变，只是表述从"待办"转"卖点"）；EN 版同步；零密钥零身份。

## 1. 模块目标

- **本模块解决什么**：把实时结果卡及相关"需人工核验"措辞，从"像派作业/半成品"改为"讲零幻觉卖点"，让评委读懂这是 H5 零虚构的严谨体现。
- **对冻结成功标准的贡献**：S4 创新/落地的对外呈现 + 把 H5「零虚构」从隐性纪律变成评委可感的正向叙事。
- **诚实性要求**：纯措辞；语义不变（实时仍不臆造、基准查询仍显示真实值）；verified 内容逐字不变。

## 2. 范围

- **本轮必须做**：
  1. **T1 · 改实时卡 manual_meta 措辞**：`i18n.py` 的 `realtime_manual_meta`（zh+en）从"需人工核验（除非已有离线核验缓存）"改为**讲价值**版（目标文案见 §6，Codex 可在 §10 内润色），核心：①坚持「零臆造」②实时新问题默认标「待核验」绝不编造 ③基准查询已 OpenAlex 核验显示真实值。
  2. **T2 · 审计其余"需人工核验"措辞**：检查点即核验空态、证据卡占位等处文案，凡读着像"派作业"的，统一改为"零幻觉/绝不臆造"价值框架（点即核验空态 `零幻觉：本地找不到可精确匹配的原句时…绝不高亮绝不猜测` 已是价值版，保留）；状态徽章 `需人工核验` 短标可保留（布局考虑），靠周边文案讲清价值。
  3. **T3 · 承重复验 + 合规 + 复盘**。
- **明确不做**：改实时是否调连接器的逻辑（仍不现场调，语义不变）；改 verified 数值/证据/标题摘要；改点即核验忠实逻辑；引框架/外链；为讲卖点而夸大或改变事实（实时确实没核实这三字段——表述转正向但不能说成"已核实"）。
- **允许修改 / 新建的文件**：
  - 措辞：`src/scholarloop/demo/i18n.py`（zh/en 文案）、必要时 `src/scholarloop/demo/design.py`（仅同步 JS 兜底 fallback 串，不改逻辑）。
  - 新建测试：`tests/test_m180_*.py`（**不改** `test_m010..m170`）。

  - **不得**改动：`demo/{realtime.py,source_text.py,interactive.py,studio.py,assemble.py,enrich_view.py}` 逻辑、`retrieval/`/`rank/`、`reports/m010..m170/**`、各判据、任何 verified 数值。
