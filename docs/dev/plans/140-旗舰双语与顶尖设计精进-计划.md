## 1. 模块目标

- **本模块解决什么**：把旗舰 Demo 从"已顶尖"再推到**无可挑剔的多角度高水准**——①**双语 i18n**（中文为主 + 中/EN 切换）；②**设计系统精进到 shadcn 级**（中性令牌 + 深色模式 + 焦点环 + 间距节奏 + 字体配对 + 一致圆角/阴影刻度）；③**实时搜索 UX 精进**（示例问题、渐进式思考态、结果卡打磨）；④**首屏/onboarding + 无障碍 + 响应式 + 克制动效**；⑤**信任叙事前置**（点即核验 + 自我证伪作为差异化突出）。
- **对冻结成功标准的贡献**：S4 落地（旗舰可信交互更强）、S3 结构化（呈现更清晰）、S5 合规（不破忠实）。
- **诚实性要求**：i18n 仅译 UI 文案，**不改任何 verified 数值/证据**；所有承重（忠实/稳定/诚实）继承不松。

## 2. 范围

- **本轮必须做（多角度，均守忠实/稳定/诚实）**：
  1. **T1 · 双语 i18n 系统**：轻量字典（`zh`/`en`）+ 中/EN 切换（URL 参数或开关，**确定性、无外部依赖**）；UI chrome（标题/按钮/标签/状态/提示/空态/错误）全可译，**中文为主**；论文内容/语料字段**保持英文原文**（忠实，不翻译证据）；关键技术术语双语并排。**i18n 不触碰 verified JSON 数值**。
  2. **T2 · 设计系统精进（shadcn 级）**：重构 `design.py` 令牌——中性色板（background/foreground/muted/border/primary/accent，明确对比）、间距阶梯（4/8 基准节奏）、字体阶梯 + 配对（sans 正文 + mono 数字/代码）、圆角刻度、柔和阴影刻度、**焦点环**；**深色模式**（CSS 变量 light/dark + 切换，尊重 `prefers-color-scheme`）；组件统一（卡片/徽章/输入/按钮/标签页/表格/骨架）。
  3. **T3 · 实时搜索 UX 精进**：**示例问题 chips**（点击填入）、**渐进式"思考"状态/骨架屏**（拆解→检索→排序可视）、结果卡打磨（标题/摘要/理由/分数/成本优雅排版）、空/错/超时态设计；诚实标注实时非 verified + 成本。
  4. **T4 · onboarding/首屏 + 无障碍 + 响应式 + 动效**：清晰 hero（价值主张 + 怎么用）、首次引导、**全状态设计（无半成品）**；可访问性（语义/ARIA/键盘/对比）；响应式（桌面/平板/窄屏）；**克制动效**（CSS 过渡、尊重 `prefers-reduced-motion`、零抖动）。
  5. **T5 · 信任叙事前置**：把**点即核验**（fabrication=0 现场可验）+ **自我证伪方法论**（M100 频率消融）作为差异化在旗舰显著位呈现。
  6. **合规 / 可复现 / 复盘**。
- **明确不做**：改 M010–M120 任何产物/源码行为/判据、M080/M120 既有面板/`render.py`/`verify.py`/`interactive.py`/`source_text.py`、M130 关系图**确定性布局算法**（可换皮不可破确定性）；引重前端框架（React/Vue/Tailwind 构建）/外部 CDN/字体外链（离线可复现）；翻译/改动任何 verified 证据数值；编造证据/实时结果；放身份信息；为观感牺牲忠实/稳定。

  - **不得**改动：FROZEN 件、`spike/**`、`reports/m010..m130/**`（数据只读）、M010–M120 源码、M080/M120 既有逻辑、各模块验收判据。

## 4. 调研结论

- **来源**：GitHub `gh search`（design-system/dashboard）、shadcn/Radix/Tremor/HeroUI、2026 UI 最佳实践、AI 信任 UX。
- **可直接落地的结论**：①高级感=中性令牌+一致刻度+焦点环+深色模式（克制）；②双语贴合中文评审+显国际水准；③全状态设计（无半成品）是顶尖铁律；④动效须克制+尊重 reduced-motion，零抖动。
- **对本计划的影响**：承重=i18n 不破忠实 + 设计精进不破稳定/忠实 + 继承 M130/M080/M120 全承重。
