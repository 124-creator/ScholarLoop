## 1. 模块目标

- **本模块解决什么**：把已 verified 的能力链——复杂查询 → 拆解（子查询+判据）→ 多源召回+可解释排序（M010）→ 10 字段证据卡 + 论文×判据证据矩阵 + 证据状态（M020）——呈现为面向研究生科研场景的**可运行 Web 智能系统**。这是官方**必交演示物料**，也是 S4 落地分与 S3 结构化分的展示载体。
- **对冻结成功标准的贡献（映射 015 S1–S5）**：
  - **S4 落地（15%）**：本模块的**主交付**——可运行 Web Demo、成本可控（默认离线预计算、0 LLM 调用）、可复现；「对接真实学术 API」以 M010 已预留的连接器接口 + 证据状态提示呈现（不实调用）。
  - **S3 结构化（10%）**：把证据矩阵 / 排序清单 / 字段卡以**列表 + 矩阵表**结构化展示，替代纯文本。
  - **S5 合规（承重）**：UI **忠实呈现** M020 JSON——逐字段显示 4 类证据状态；作者/年份/来源/DOI 显示「需人工核验 + 在线连接器」，**绝不在页面伪造**任何字段。
  - **S1（F1·70%）/ S2（效率·20%）不在本模块**：F1 由 M010 结清；本模块默认离线、不引入新的 F1/效率声称，只如实展示 M010 的效率账。

## 2. 范围

- **本轮必须做**：
  1. **轻量后端 API**（复用现有 pipeline / evidence 层，不另起逻辑）：列出基准查询（预计算）、取某查询的证据文档、健康检查；**预计算优先**。
  2. **预计算优先的数据服务**：直接服务 `reports/m010/results.json` 与 `reports/m020/evidence/*.json`；基准集**即时、可复现、每请求 0 LLM 成本**。
  3. **极简前端**（服务端渲染 HTML 或原生 JS，不引重框架）：查询输入/选择 → 拆解（子查询+判据）→ 排序论文（score + reason）→ 证据矩阵表（论文×判据，✅ + 可定位片段）→ 逐篇 10 字段卡（**证据状态醒目**）。
  4. **忠实呈现保证**：页面/接口渲染的字段**逐字等于** M020 JSON，不新增、不臆造；不可落地字段按其状态展示。
  5. **成本/复现护栏**：实时模式默认关闭或限速限额；离线基准模式为默认；确定性。
  6. **跨域演示**：含 LitSearch 样例 + ≥1 个碳价/AI 科研演示查询（S4 泛化）。
  7. **可复现跑 + 测试 + 复盘**。

## 4. 调研结论

- **来源**：`00-问题描述.md`（§技术形态要求 Web 智能系统、§比赛展示成功标准、§out-of-scope「不做只返回标题列表 / 不做只好看缺证据链」）、`015` S3/S4、`035` §4 边界、M010/M020 verified 产物。
- **可直接落地的结论**：
  1. **预计算优先**最契合「成本可控 + 可复现」：基准集直接读 JSON，0 LLM 成本、确定性；实时模式作为**可选、限额**的加分项，默认关闭。
  2. **忠实呈现是 S5 在 UI 层的延伸**：渲染层只能照搬 M020 JSON 的字段与状态，做成可机检（渲染负载 == 源 JSON 子集）——把「页面不伪造」从口号变成断言。
  3. **轻栈优先**：Python 后端（FastAPI/Flask 任一）+ 服务端模板/原生 JS，单命令启动，无重前端框架——符合个人参赛「可运行、可复现、成本可控」，也避免视觉喧宾夺主（035 §4）。
- **对本计划的影响**：W2（忠实呈现零伪造）+ W1（可运行）是承重；视觉打磨为辅，不得越过证据链优先级。

## 11. 风险、边界与回滚

- **边界风险（035 §4 红线）**：Web 化易滑向「页面好看但缺证据链」。缓解 = 承重验收锁定在「可运行 + 忠实呈现零伪造 + 证据状态可见」，视觉为辅；out-of-scope 明确不做鉴权/部署/重框架。
- **忠实呈现风险**：UI 可能美化或隐藏「证据不足/需人工核验」。缓解 = T3/T6 自动断言渲染负载 == 源 JSON、状态徽标必现。
- **实时模式成本风险**：默认关闭、限额；做不到有界就只留离线模式并上报。
- **回滚**：本模块独立于 M010/M020 与 FROZEN；失败仅废弃 `src/scholarloop/web/**` 与 `reports/m030/**`，不污染上游 verified 证据。
