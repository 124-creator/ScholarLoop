## 1. 模块目标

- **本模块解决什么**：把已 verified 的四层能力——A-v2 检索排序（M040）、B-lite 证据矩阵（M020）、真实学术 API 富化（M050）、研究空白发现（M070）——整合进**一个可单命令运行的 Web 闭环演示**，并**收束官方初赛提交物料**，让作品的硬核（七承重墙）以**忠实、可复现、零夸大**的方式对外呈现。
- **对冻结成功标准的贡献**：
  - **S4 落地（15%）**：端到端可运行系统，对接真实学术 API（M050），覆盖官方 3.1 全部核心功能（查询理解分解 / 迭代检索 / 综合排序 / 结构化归纳）。
  - **S3 结构化（10%）**：列表 + 关系/矩阵展示（证据矩阵 + 研究空白关系），对齐官方「列表、关系图」要求。
  - **S4 创新（15%）/ 泛化（10%）**：提交文档把 M070 研究空白可证伪性、M060 跨基准泛化作为差异化叙事，**每条主张挂 verified 证据**。
  - **S5 合规 / H5**：整合页面忠实呈现（值逐字等于后端 verified JSON），真实连接器值带出处、未解析项标「需人工核验」，研究空白叙述零虚构。
- **诚实性要求**：提交物料**不得做出任何超过 verified 证据的主张**；演示**不得编造**任何数据；作品材料**不得含学校/学院/导师等身份信息**（官方公平性红线）。

## 2. 范围

- **本轮必须做**：
  1. **整合 Demo 层（新建，不改 M030 `web/`）**：单命令起服务，把四层能力整合到一个查询的端到端视图——①查询拆解（M010/查询层）②A-v2 排序结果（M040 `reports/m040`，区分高度/部分相关）③逐条证据矩阵（M020）④真实连接器富化的作者/年份/DOI（M050 enriched，带出处；未解析标「需人工核验」）⑤研究空白发现（M070 列表 + 关系矩阵）。
  2. **整合忠实呈现校验（承重）**：每个面板的展示值**逐字等于**其后端 verified JSON（A-v2 排序、M020 证据、M050 enriched、M070 gaps）；零伪造、零越界、缺失字段显式标注不补写。
  3. **效率/成本呈现（S2）**：页面/文档透明展示推理成本口径（API 调用、token、延时，复用 M010/M040/M060 已记账数据），对齐官方 20% 效率维度。
  4. **官方提交物料收束**：①**作品简介**（≤300 字，中文，无身份信息）；②**项目文档**（标准模板式详述：问题/方法/系统架构/创新点/落地/泛化/评测结果/复现/合规）；③**项目视频脚本 + 演示走查脚本**（脚本与分镜，**实际录制由人类完成**，本模块只交可执行脚本）；④**指标汇总表**（跨两基准 F1/效率/结构化，全部引 verified 产物）；⑤**复现说明**（单命令运行 + 数据源/许可 + 冻结配置 sha）；⑥**创新叙事**（研究空白可证伪性 + 跨基准泛化，挂证据）。
  5. **携带项收束**：把历史携带项在提交材料里处理干净——(a) 把「BGE-base」措辞订正为实测 **bge-small-en-v1.5**；(b) M040 cross-encoder 边际贡献据 NDCG 给出保留/可砍的诚实说明；(c) M070 频率混杂——主张精确表述为「(高活跃·零历史共现)组合空白填补率显著高于随机概念对」，**可选**补一条频率配平基线消融；(d) M060 cross-encoder 在第二基准未单独消融的诚实标注；(e) M050 礼貌 UA 邮箱在提交版做脱敏（不泄身份）。
  6. **合规 / 可复现 / 复盘**。
- **明确不做**：改 M010–M070 任何产物/源码行为/验收判据/A-v2 配置（**全冻结只读**）；改 M030 已 verified 的 `web/`（整合层**新建**在 `src/scholarloop/demo/`）；做出**超过 verified 证据的任何指标主张**；编造演示数据；引重型前端框架（守 035 §4 视觉不喧宾夺主红线）；鉴权/部署基建；重训/重评（不重跑 F1 承重墙，只**引用** verified 数值）；在材料中放学校/导师/个人身份信息。
- **允许修改 / 新建的文件**：
  - 新建：`src/scholarloop/demo/**`（整合应用 + 数据装配 + 渲染 + 忠实校验）。
  - 新建提交物料：`docs/submission/**`（作品简介、项目文档、视频脚本、指标汇总、复现说明、创新叙事、数据源/许可汇总）。
  - 新建测试：`tests/test_m080_*.py`（**不改** `test_m010..m070`）。
  - 新建产物：`reports/m080/**`（忠实校验审计、烟测、复现核对、secret_scan、validation_summary）。
  - **不得**改动：FROZEN 件、`spike/**`、`src/scholarloop/web/**`（M030 冻结基线）、`reports/m010..m070/**`、M010–M070 既有源码行为、各模块验收判据。

## 4. 调研结论

- **来源**：官方赛题 §3/§4/作品要求、015 成功标准、M010–M070 verified 产物与复盘、PaSa/SPAR/Ai2 Paper Finder 对标（赛题 §7，用于创新叙事定位差异化）。
- **可直接落地的结论**：
  1. 七个 verified 模块已覆盖官方核心功能 + 三评分维度，M080 的价值在**整合呈现 + 可信收束**，不在再造能力。
  2. 差异化叙事应锁定**可信/可证伪**：M020 零伪造证据链 + M050 真实出处 + M070 可证伪研究空白 + M060 跨基准泛化——对标系统多强调 Recall/F1，本作强调**可复核证据链 + 研究空白可证伪**。
  3. 提交物料的最大风险是**夸大**——必须每条主张挂 verified 产物，宁缺毋虚。
- **对本计划的影响**：承重转为「忠实呈现 + 零夸大可溯源」；整合层走 M030 的 offline 0-LLM 忠实渲染范式以保确定性与零伪造。

## 11. 风险、边界与回滚

- **核心风险（夸大）**：提交文档易在措辞上**超出 verified 证据**（如把单基准结果说成普适、把携带项忽略）。**红线** → 每条主张挂 verified 引用，携带项如实写入，宁缺毋虚。
- **忠实呈现红线**：整合页面值逐字等于后端 verified JSON，缺失标注不补写，零伪造。
- **身份合规红线**：材料不得含学校/学院/导师；M050 UA 邮箱提交版脱敏。
- **冻结红线**：M010–M070 + M030 `web/` 全部只读；整合层新建。
- **回滚**：本模块为新增整合层 + 文档；失败仅废弃 `src/scholarloop/demo/**`、`docs/submission/**`、`reports/m080/**`，不污染任何 verified 证据。
