## 1. 根因

`src/scholarloop/retrieval/dense_v2.py` 的 `_model_version()` 使用了：

```python
getattr(self.model, "get_sentence_embedding_dimension", self.model.get_embedding_dimension)()
```

第三个参数会被 Python eager 求值；当当前 `sentence-transformers` 环境不存在旧接口
`get_embedding_dimension` 时，实时路径在现场构造 `DenseV2Retriever` 即抛 `AttributeError`。
离线 M040 评测使用已落盘缓存，过去未触发该分支。

## 2. 修复

- 将维度获取改为惰性兼容：
  1. 优先 `get_sentence_embedding_dimension`；
  2. 缺失时再取 `get_embedding_dimension`；
  3. 两者都缺失才显式报错。
- 未修改 `_fingerprint()`、`_paths()`、A-v2 权重、候选池、重排、缓存 stem 或任何评测 JSON。
- 新增 `tests/test_m150_realtime_hotfix.py`，锁住：
  - 新旧 getter 兼容；
  - `_model_version()` 字符串仍为 `dim=384`；
  - dense 缓存键仍为 `dense_v2_BAAI_bge_small_en_v1_5_11bddb2dab1c0d65.*`；
  - M040/M060 F1 与 M040 A-v2 逐查询排序摘要不变。
