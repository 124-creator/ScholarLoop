# 015 · 成功标准校准件（对齐官方评测口径）

- 性质：`010-问题定义.md`（FROZEN）的**校准件 / amendment**
- 状态：FROZEN（2026-06-18，本会话经 Router 授权；见 `docs/dev/000-决策日志.md` ADR-002）
- 生效方式：在**冲突处**优先于 `010-问题定义.md` 的成功标准 2 / 3；其余条款仍以 010 为准
- 上游证据：`00-赛题/科研场景下复杂学术查询的智能论文搜索与推荐.txt`

> 本件只校准「成功标准的权重与证据来源」，**不改变**冻结目标、技术形态、硬约束与 out-of-scope。

---

## 1. 为什么需要校准

`010` 把官方评测口径列为「未知项」，但赛题原文已明确：

| 维度 | 官方权重 | 评测方式 |
|---|---|---|
| **F1 Score** | **70%** | 公开测试集（30%）+ **隐藏测试集（30%）** 自动评分 |
| 运行效率 | 20% | API 调用次数、Token 消耗、端到端延时 |
| 结构化展示 | 10% | 是否用列表 / 关系图等组织结果 |
| 专家评分 | （成绩 40%） | 创新性 15% + 落地可行性 15% + 算法泛化性 10% |

> 成绩构成：竞赛得分 60%（= 自动评分项按 F1/效率/结构加权）+ 专家评分 40%。

与 `010` 原成功标准的错位：原标准把「自建 50 题集 + 人工合理率 ≥85% + 证据链/研究空白」当承重创新，
而官方 70% 压在 **F1 + 隐藏测试集**——`02-评审分级.md` 的 X1（自建自评利益冲突）/ X2（基线不可比）
正是这一错位的致命体现。

---

## 2. 校准后的成功标准（按官方权重重排）

### S1 · F1 为承重墙（官方 70%）— 用公开基准做金标准

- **金标准来源改为公开学术检索基准**，优先级：
  1. **LitSearch**（EMNLP 2024，arXiv:2407.18940）— 学术文献检索 retrieval benchmark；
  2. **PaSa / RealScholarQuery**（参考数据集 `github.com/bytedance/pasa`）；
  3. **AstaBench / PaperFindingBench**（`github.com/allenai/asta-bench`，arXiv:2510.21652）。
- 用公开基准的官方相关性判定（ground truth）计算 **F1 / Recall@20 / Precision@10**，
  **不再以参赛者自标注作为对外声称「优于基线」的唯一证据** → 化解 X1。
- 自建 50 题集**保留但降级**为：开发期 dev 集 + 案例演示集 + 错误分析集，不进入对外指标声称的主链路。
- 阈值：在选定公开基准上，ScholarLoop 完整流程的 **F1 明显高于** 同候选池下的基线（关键词 / BM25 / Embedding / 单轮 LLM）。

### S2 · 运行效率升为一等约束（官方 20%）

- 必须记录并报告：**平均 API 调用次数 / 每查询 Token 消耗 / 端到端 P50·P95 延时**。
- 设计期即设预算上限（见 `040` C-lite 闸门）；多轮自我修正臂必须证明「效率代价换来的 F1 提升」为正。

### S3 · 结构化展示（官方 10%）

- 输出必须用**列表 + 关系图 / 证据矩阵**等结构化形式，而非纯文本段落。
- 证据矩阵层（B-lite）在此处承担官方 10% 的结构化得分，**不再与 F1 抢承重位**。

### S4 · 专家分差异化（官方 40%：创新 15 + 落地 15 + 泛化 10）

- **创新（15%）**：复杂查询拆解、多轮检索与自我修正、综合排序、证据矩阵化、研究空白发现。
- **落地（15%）**：可运行 Web Demo、成本可控、对接真实学术 API、可复现。
- **泛化（10%）**：跨学科多案例（碳价 / AI 科研智能体 / 跨学科），证明非单领域过拟合；
  隐藏测试集表现是泛化的硬证据。

### S5 · 合规（不变，仍以 010 为准）

不虚构论文 / DOI / 作者 / 期刊 / 年份；显式区分「已有证据支持 / 证据不足 / 存在争议 / 需人工核验」；
对低置信度推荐提示人工核验。

## 3. 校准对 010 各条款的处置

| 010 原条款 | 处置 |
|---|---|
| 成功标准 2「自建 50 题 + ≥2 指标优于基线 + 人工合理率 ≥85%」 | **被本件 S1 取代**：金标准改用公开基准，F1 为主；人工合理率降为辅助证据 |
| 成功标准 3「10 项证据字段 + 证据状态区分」 | **保留**，但定位为 S3（结构化 10%）+ S4（创新）载体，非 F1 承重 |
| 成功标准 1 / 4 / 5（功能 / 创新 / 交付） | **保留**，并入 S3 / S4 |
| 硬约束、out-of-scope、技术/数据/合规约束 | **完全保留，不改动** |

## 4. 不做的事（防止校准被误用为重新开题）

- 不借校准改变冻结目标、赛题方向、Web 形态或一等奖定位；
- 不取消评测集、基线对比、消融实验；
- 不把碳价预测变成唯一核心场景；
- 若需改动上述任一，必须回到 `010` 重新冻结并追加新的 ADR。
