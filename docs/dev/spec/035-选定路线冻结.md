## 1. 选定路线

> **主干 = 路线 A（评测驱动的可解释检索排序）**
> **差异化件 = 有界证据矩阵层（B-lite）+ 一个自我修正消融臂（C-lite）**

这不是第四条路线，而是「A，经过最小增强」。它不改变冻结目标与候选集，只回答「先建什么、用什么补创新」。

### 1.1 为什么是 A 主干

- 唯一一条让个人参赛者在时限内**几乎确定**交齐官方自动评分物料（F1 / 效率 / 结构化）与可复现实验的路线；
- 评测 / 基线 / 消融是其原生产物，X1–X3 补齐后指标最经得起复现质询；
- 失败模式是「优雅降级」——即便创新不够，仍是能跑、能测的系统，不会归零。

### 1.2 为什么强制叠加 B-lite + C-lite

- 纯 A 唯一硬伤是创新关（官方专家分创新 15%，评委一句「这不就是 RAG」即危）；
- B 的证据矩阵层与 C 的一个消融臂都是 A 流水线的**自然下游**（先检索排序，才谈得上矩阵化与「补检是否有用」），正好补上 A 的硬伤；
- 二者**有界**：B-lite 只做证据矩阵 / 结构化展示，不做大规模交互图谱；C-lite 只做一个「单轮 vs 多轮+质检」消融臂，不做无限循环代理。

---

## 2. 建造顺序（冻结）

1. **A 流水线**：查询拆解 → 多源召回 → 可解释综合排序 → 证据抽取。
2. **证据矩阵层（B-lite）**：把推荐结果组织成结构化证据矩阵（承接官方结构化 10% + 创新）。
3. **自我修正消融臂（C-lite）**：单轮 vs 多轮+质检的 go/no-go 消融。
4. **（条件触发）图谱 / 研究空白层**：仅当 `030` §4.1.3 的 schema spike 通过且时间充裕。

## 3. 进入验证桩的阻断集（must-fix，全部见 `040-验证桩规格.md`）

### 共性（X1–X3）
- **X1** 评测金标准 / 相关性判定协议 →（本项目用**公开基准**作金标准，见 `015` S1）；
- **X2** 基线对比协议（同语料 / 同候选池 / LLM 虚构如何计入）；
- **X3** 数据来源点名（OpenAlex / Semantic Scholar 等）+ 许可 / 字段 / 频率核验。

### 路线 A 特有
- **A-1**（030 §3.1.1）数据充分性 spike：摘要 / 开放片段能否支撑 10 项证据字段真实归因；
- **A-2**（030 §3.1.2）证据归因准确率操作化为可测指标；
- **A-3**（030 §3.1.3）创新差异化件锁定（已由本件锁定为 B-lite + C-lite）。

### C-lite 携带闸门
- C-lite 必须通过 go/no-go 消融（多轮+质检在 ≥1 主指标上**真实且可复现**优于单轮），否则砍掉 C-lite，保留纯 A 主干。
