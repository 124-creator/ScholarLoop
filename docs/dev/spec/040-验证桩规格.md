## 1. S-X3 · 数据来源充分性

- **目标**：点名并核验公开学术数据源，证伪「公开元数据+摘要+开放片段够用」这一全局假设。
- **候选来源（须自行核验，不预先断言）**：OpenAlex、Semantic Scholar（API）、Crossref（DOI 解析）、arXiv（开放全文片段）。
- **输入**：从公开基准（LitSearch / RealScholarQuery）取 **5 个复杂查询**对应的真论文集合。
- **动作**：
  1. 核验每个来源的**许可、字段覆盖（标题/作者/年份/DOI/摘要/引用/开放全文链接）、速率限制**，成文记录；
  2. 对 5 个查询的目标论文，实际拉取元数据 + 摘要 + 可得开放片段；
  3. 人工确认：仅凭拉到的内容，能否填出 10 项证据字段（标题 / 作者年份 / 来源·DOI / 推荐理由 / 支持问题 / 方法 / 数据 / 结论 / 局限 / 关联强度）。
- **通过判据（全部满足）**：
  - 至少 **1 个主检索来源 + Crossref 做 DOI 校验**可用且许可允许；
  - 5 个查询的目标论文，**≥80%** 能取到摘要；
  - 10 项证据字段中，**方法 / 数据 / 结论** 三项的可填率 **≥70%**（局限项允许标「证据不足」）。
- **失败处理**：若可填率不足，证据矩阵是空壳 → 退回 `015`，把证据字段缩减为可支撑子集，或改取开放全文来源。
- **不确定度**：中。
- **产出**：`spike/x3-数据来源核验.md` + 5 查询取数样本（保存原始响应）。

## 2. S-X1 / S-X2 · 评测协议小样本走通（含基线对比）

- **目标**：证伪「自建自评指标可信」漏洞——改用公开基准金标准，在同候选池下走通一次端到端评测。
- **金标准**：用 LitSearch（或 RealScholarQuery）**官方相关性判定**，不用参赛者自标注（化解 X1 利益冲突）。
- **输入**：公开基准的 **10 个查询** + 其官方 ground truth。
- **动作**：
  1. **固定受控协议**：同一公开语料 / 同一候选召回上限 K（如 Top-100）/ 所有系统共享候选池（pooling）；
  2. 跑 4 个基线 + ScholarLoop-min（仅 A 主干最小版）：关键词、BM25、Embedding、单轮 LLM、A-min；
  3. **单轮 LLM 基线的虚构处理写清楚**：虚构论文（无法在来源解析 DOI）计为不相关，不得因虚构而占便宜也不得豁免；
  4. 计算 Precision@10 / Recall@20 / F1，输出每系统一行结果；
  5. 抽 2 个查询做**第二判定来源**核对（第二人或外部锚点），报告一致性。
- **通过判据（全部满足）**：
  - 评测脚本可端到端跑通并复现（同输入两次结果一致）；
  - 4 基线 + A-min **均在同候选池**上评分，协议成文；
  - 至少在该 10 题小样本上，**A-min 的 F1 不低于最弱基线**（仅为「协议走通+方向可行」的弱判据，不作最终性能结论）；
  - 第二判定一致性可报告（哪怕偏低，须如实记录）。
- **失败处理**：协议跑不通或基线不可比 → 退回固定语料/候选池定义，不进入模块实现。
- **不确定度**：中高。
- **产出**：`spike/x1-评测协议走通.md` + 评测脚本 + 结果表 + 受控协议成文。

## 3. S-A1 · 证据归因充分性

- **目标**：A 自报的最危险假设——摘要/开放片段能否支撑**真实**归因（而非看似合理的编造）。
- **输入**：S-X3 的 5 查询目标论文中，抽 **10 篇**。
- **动作**：对 10 篇填 10 项证据字段，并**逐字段标注来源出处**（摘要某句 / 开放片段某段 / 元数据字段）；无出处的字段必须标「证据不足/需人工核验」，禁止臆测。
- **通过判据**：
  - 方法 / 数据 / 结论字段中，**有明确出处** 的比例 **≥70%**；
  - **0 处** 虚构 DOI / 作者 / 年份 / 期刊（任一虚构即整桩 FAIL）；
  - 无出处字段全部如实标注为证据不足，无「看似合理实则编造」。
- **失败处理**：归因率不足 → 把「证据矩阵」定位降级为「带出处的部分字段 + 显式缺口」，并在 `015` 记录。
- **不确定度**：中。
- **产出**：`spike/a1-证据归因.md`（10 篇逐字段出处表）。

## 4. S-A2 · 证据归因准确率操作化

- **目标**：把「证据归因准确率」从口号变成可测指标（否则不能用来声称优于基线）。
- **动作**：定义 (a) 判定单位（每篇·每字段）；(b) 正确定义（字段内容与出处一致且不外推）；(c) 抽样量（≥30 个字段实例）；(d) 判定者与利益冲突缓解（抽样交第二人复判）。
- **通过判据**：指标定义成文、可复算、样本量与判定流程明确，且在 S-A1 的 10 篇上试算出一个数值。
- **不确定度**：低。
- **产出**：`spike/a2-归因准确率定义.md`。

## 5. S-C · C-lite go/no-go 消融闸门

- **目标**：证明「多轮+质检」相对「单轮」有**真实且可复现**的提升，否则砍掉 C-lite。
- **动作**：
  1. 在 S-X1 的 10 题上跑三臂：单轮 / 多轮 / 多轮+确定性质检（DOI 在 Crossref/OpenAlex 可解析、作者年份匹配）；
  2. **质检为确定性校验，非 LLM 自评**；
  3. 温度 0 + 可固定随机种子；每臂跑 **N=3 次**，报告 mean±std；记录 API 次数 / Token / 延时。
- **判据（go）**：多轮+质检在 ≥1 主指标（F1 或 Recall@20）上 **mean 提升为正且方差不掩盖提升**，且效率代价在 `015` S2 预算内。
- **判据（no-go）**：无提升或提升被方差吞没 → C-lite 翻 RED，保留纯 A 主干，写入 ADR。
- **级别**：本 spike 不阻断验证桩 PASS，但结论必须明确（go 或 no-go）。
- **不确定度**：高。
- **产出**：`spike/c-消融闸门.md` + 三臂结果（mean±std + 效率）。

## 6. Spike 执行边界（给执行方）

- 所有 spike 只用小样本（5–10 查询），**禁止借 spike 搭半成品业务系统**；
- 所有原始 API 响应、脚本、结果表必须保存到 `spike/`，可复查可复现；
- 命中以下情形立即停机上报，不自行扩范围：来源许可不允许、字段覆盖不足、需要付费全文、需要新依赖/新架构、与本规格判据冲突；
- spike 完成后产出 `docs/dev/spec/040-验证桩结论.md`（PASS/FAIL + 逐 spike 证据），由总指挥用新鲜上下文初查。
